# AI 编码工作台页演练

## 原始需求

为桌面端 AI 编码产品设计一个“项目工作台页”。用户打开产品后，应能一眼看清当前项目、正在进行的任务、下一步可做之事，并能快速发起新任务或恢复旧任务。页面需兼顾高级感与工具效率，不做营销式首屏。

约束：

- 主场景是开发者在工作中高频进入，不是首次访问宣传页
- 需适配宽屏主窗口与窄栏模式
- 需考虑无项目、索引中、任务进行中、任务失败、只读仓库五类常见状态
- 视觉对标取 Apple 的层级与和谐、Google Material 的系统化与自适应、标杆站的构图节奏与记忆点，但不得抄皮相

## 第 1 稿设计草案

### 今判

先以工具型工作台定骨架，再补一个可记忆的轻品牌记号；不做 hero，不靠特效堆高级感。

### 所求

- 三秒内看懂当前项目状态、任务进度与下一步操作
- 发起新任务与恢复旧任务都在首屏可达
- 页面像成熟桌面工具，不像通用 SaaS 模板

### 所重

- 扫读效率
- 首屏层级
- 状态反馈
- 视觉克制
- 跨尺寸稳定

### 行为校准

- 目标用户：日常在 IDE、终端与浏览器间切换的工程师
- 关键场景：早晨恢复上下文；切项目后确认环境；任务跑到一半回来接着做；失败后重新进入排查
- 触发动作：打开产品首页；切换项目；点“新任务”；点“恢复任务”；点失败卡片
- 系统反馈 / 预期结果：首屏直接显示项目身份、任务队列、可执行入口与环境状态；不需先读说明
- 页面 / 模块范围：工作台首页，不含深层设置与历史全量列表
- 关键流程：查看项目身份 -> 判断当前进度 -> 发起或恢复任务 -> 进入具体会话
- 状态矩阵：无项目、索引中、就绪、任务运行中、任务失败、只读仓库、窄栏压缩
- 对标标尺 / 参考系：Apple 的 hierarchy / harmony / consistency；Material 的 system / token / adaptive / motion；标杆站只借留白节奏与记忆点
- 视觉 / 气质方向：石墨灰中性基底 + 矿物蓝强调；像精密工作台，不像宣传页
- 审美质量 / 视觉层级：主操作一眼可见；项目身份与任务流同层；辅助信息退后
- 排版 / 留白 / 对比：大标题不用超大；用字阶、留白和浅材质分层；高对比只留给主操作与关键告警
- 动效 / 材质 / 克制边界：只用轻位移与淡入解释状态切换；不用大面积毛玻璃和厚阴影
- 信息密度 / 留白 / 响应式：宽屏三列，窄栏单列折叠；状态与操作始终首屏可见
- 现有 UI 基线 / design system：假定已有按钮、输入框、卡片、进度、状态标签与列表组件可复用
- 验收口径：三秒看懂主次；关键状态不缺；主 CTA 唯一；窄栏不丢主路径
- 现有约束：高频工具页；数据刷新频繁；弱网与长任务并存

### 页面骨架

1. 顶部项目带：左侧项目名、分支、仓库状态；右侧全局搜索与设置入口。
2. 首屏主区：左 7 份为“继续工作”，展示当前任务、下一步建议、恢复按钮；右 5 份为“项目健康”，展示索引、测试、改动摘要。
3. 次级区：最近任务列表、模板入口、知识入口三块并列。

### 视觉骨架

- 栅格：12 栏；主内容区 7/5 分栏；窄屏改单列
- 间距阶：4 / 8 / 12 / 16 / 24 / 32
- 字阶：13 / 15 / 18 / 24 / 32
- 颜色角色：石墨灰为底；矿物蓝为主强调；琥珀作警示；红仅用于失败态
- 圆角 / 阴影：圆角中等；阴影只留一级；卡片靠边框与层次分隔

### 状态口径

- 无项目：主区不显示空卡海，而给“打开仓库”与最近项目列表
- 索引中：项目健康区显示阶段、剩余时间感知与可先做动作
- 任务运行中：继续工作区固定显示进行中任务与取消/查看细节
- 任务失败：失败卡片上移到主区，给复试与查看日志
- 只读仓库：隐藏会写仓库的 CTA，改成只读分析路径
- 窄栏：项目健康降为摘要行，最近任务折叠为抽屉入口

## 复核后定案

### 结构收敛

1. 首屏只留三块：项目身份带、继续工作主区、项目状态侧栏。
2. “模板入口”“知识入口”收进右上角“工具”菜单，不再并列占首屏。
3. 最近任务列表下沉到第二屏首块，但保留一条最近任务摘要挂在主区底部。

### 视觉收敛

1. 主区只保留一个强调按钮“继续当前任务”或“新建任务”，二者不并列同级抢焦点。
2. 全页强调色只给主 CTA、进行中状态与关键告警；其余状态退回中性色加标签。
3. 记忆点改为顶部项目带上的细窄活跃光带，而非额外装饰卡片。

### 状态补齐

1. 空任务列表：主区底部改为“尚无任务”说明 + “从模板开始”次级入口。
2. 任务完成：主区标题切为“已完成”，右侧给“查看结果”和“继续追问”两种后续动作。
3. 失败与只读态保留上移规则：只要阻断主路径，就提升到主区最上层。

### 最终页面说明

- 顶部项目带：项目名、分支、同步状态、工具菜单、搜索。
- 主区：一张横向主卡，固定显示当前最重要任务与唯一主 CTA；主卡下方只放一条最近任务摘要。
- 侧栏：状态行列表，依次是索引、测试、改动摘要、权限状态。每行都复用统一组件，不出现大拼装卡。
- 第二屏：最近任务列表；若为空，则显示模板与文档入口。

### 最终验收

1. 三秒内能看出项目身份、当前任务和唯一主操作。
2. 无项目、索引中、运行中、完成、失败、只读、窄栏七态都有落点。
3. 去掉颜色后，主次仍成立。
4. 首屏无第二个与主 CTA 竞争的高饱和强调块。
5. 相邻页面沿用同一套按钮、状态标签、摘要行和卡片边界规则。
