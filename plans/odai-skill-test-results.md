## 本轮已做的修正

### 源技能修正

- 收紧宿主提问工具兜底：只有当前宿主确未暴露提问工具，才允许文字兜底。
- 明确“先做后判需求 / 先改后问 / 跳过澄清或授权”属流程硬冲突，直接拒绝，不降级成普通补问。
- 明确模块、agent、模型、执行方、提供方与环境能力只能报真实命中与真实调用，禁止写死、补写、猜写或伪报。
- 明确职责/审批/权限矩阵、公共 API/字段/兼容层、默认多 agent / 第二视角 / 全库审查 / 统一输出结构 / 中途播报等全局契约改动，先问适用面、例外、成本、回滚与验收。
- 明确用户若只要草案/只停分析，但任务对象或完成态仍不清，先问清再停。
- 明确 `skill-author` 对“新增模块草案”“全局默认契约改动”须先问模块名/类型/路径/适用面/例外，不直接落文。
- 明确 `review-sslb` 若欲把默认范围改为全库扫描，先问范围、性能代价、例外与停止条件。
