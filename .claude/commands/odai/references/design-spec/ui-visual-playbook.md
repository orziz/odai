# UI Visual Playbook

## 何时读取

命中以下任一主题时，读本文件：

- 网站首页、品牌页、工作台、控制台、仪表盘、设置页、表单流、编辑器壳层
- 现有网站或应用的改版、翻新、视觉提质、去模板味、旧页面重设计
- 用户明要更高级、更像成品、更不像模板站或通用 SaaS
- 题目主矛盾不止在“美不美”，而在页面骨架、构图节奏、排版气质与反俗套
- 现有界面显居中大标题配三卡片、卡套卡、强调过多、伪技术小标签过密、首屏拥堵

若同时命中高审美提质，再同读 `odai/references/design-spec/aesthetic-benchmark.md`。

## 先判任务性质

先断这是哪种前端题：

1. 新建前端：从零或近零起页，主问题是视觉方向、页面骨架、体验节奏与实现落法。
2. 现有项目重设计：已有代码、已有组件、已有样式或已有页面模式，主问题是先查旧病灶，再做薄改升级。

两者不可混做。新建前端先立方向；现有项目先审现状。若把重设计当新建稿处理，最容易在实现层直接重写；若把新建题当修补题处理，又会落回旧模板。

## 先判页面体裁

先断当前页面属于哪一类，再谈风格：

1. 品牌页：设计本身即卖点，可放大构图、留白、记忆点与情绪张力。
2. 工具页：设计服务任务完成，优先扫读、状态、反馈、连续操作与低认知负担。
3. 混合页：先断首要任务；若用户进来先做事，按工具页；先进来先被说服，按品牌页。

未先定体裁，极易把营销页语言误套进工具页，或把工具页做成无个性的中性壳。

## 设计宣言四问

新建前端题，出草案前先用四问定一版视觉宣言：

1. 它解决什么问题，谁来用。
2. 它要站在哪种气质极点：极简、社论、工业、温润、奢华、玩具感、野性、理性，或别的清晰方向。
3. 技术、性能、可访问性与实现成本边界是什么。
4. 用户离开页面后，唯一会记住什么。

方向要鲜明，不要温吞折中。大胆不等于乱，克制也不等于平。关键是立场清楚，并把它执行到底。

## 三拨盘

出草案前，至少先给三项拨盘口径：

1. 构图张力 `1-10`
   - `1-3`：对称、稳、保守，适合设置页、后台基础页。
   - `4-7`：允许偏移、错位、大小对比与非均匀留白，适合大多数高质量产品页与工作台。
   - `8-10`：强非对称、显著负空间、章节节奏更戏剧化，只在品牌页或低频展示面慎用。
2. 动效分量 `1-10`
   - `1-3`：仅状态反馈、轻 hover、轻位移。
   - `4-7`：允许阶段入场、顺序揭示、空间切换与少量连续动效。
   - `8-10`：滚动编排、叙事转场、复杂互动；高频工具页默认不取高值。
3. 信息密度 `1-10`
   - `1-3`：疏朗、展陈感强，适合品牌与叙事页。
   - `4-7`：日常产品默认区间，清楚且不挤。
   - `8-10`：高密工具、监控、交易、运营台；此时优先行分组、分隔线与字阶，不靠大卡片装箱。

三拨盘先于细节风格。拨盘不定，后续排版、容器、动效与状态会互相打架。

## 成品感五层

不论新建还是重设计，视觉判断按五层压实：

1. 任务层：首屏先让用户知道在何处、能做什么、下一步点哪里。
2. 信息层：标题、说明、数据、状态、危险信息和次级动作有明确先后。
3. 系统层：栅格、字阶、间距、色彩、圆角、边框、阴影和动效能归入少量规则。
4. 触感层：hover、active、focus、selected、disabled、loading、empty、error、success 与回退反馈齐。
5. 记忆层：只留一个与业务相关的视觉特征；不是渐变、玻璃、噪点或大圆角的通用皮。

## 现有项目重设计四步

命中重设计题时，先走四步，不直接画新壳：

1. 扫现状：读代码与页面，识别框架、样式体系、组件模式、token、邻页语法与可复用面。
2. 断病灶：按下方审计面列出泛模板味、弱层级、缺状态、不一致与可薄改点。
3. 清遗留：列出应随本轮一并删除的旧容器、旧变体、旧 token、旧样式分支、旧文案、旧截图、旧说明与旧交互残片。
4. 做薄改：沿既有技术栈与样式系统修，不借题改框架、不整页推翻、不平地起第二套体系；新方案一旦替代旧方案，同轮同界内删净旧轨。

目标不是重做一版 Dribbble 图，而是让现有产品在不伤功能的前提下明显变好。

## 遗留清理清单

重设计题除审美提质外，还要查是否残留以下旧物：

1. 已废弃的容器壳、卡片壳、包裹层与仅为旧布局服务的间距补丁。
2. 已被新组件替代的旧变体、旧按钮样式、旧标签形态与旧导航分支。
3. 不再使用的 design token、颜色别名、阴影变量、半废字号阶与圆角阶。
4. 新旧并存的状态样式：新 loading 已上，但旧 spinner 还挂着；新 empty 已有，但旧空白占位还留着。
5. 旧文案、旧占位图、旧截图、旧 demo 数据、旧 favicon、旧 meta 与过期设计说明。
6. 为兼容历史视觉临时加的特判 class、布尔开关、废 props、废 story、废测试快照。

原则：能在本轮替掉的，不留到下轮；能同界删净的，不留新旧双轨；跨文件删不稳者，明记风险与待迁清单，不假装已清。

## 重设计审计面

现有项目至少审以下几层：

1. 排版：是否仍是默认字体、字重层级贫乏、正文过宽、数字未等宽、标题无气场、字距未调。
2. 颜色与表面：是否混冷暖灰、强调色过多或过饱和、紫蓝 AI 渐变成习惯动作、阴影全是通用黑影。
3. 布局：是否处处居中、三等宽卡片成瘾、无最大宽度、分栏靠脆弱 flex 算式、圆角和 padding 全页一刀切。
4. 状态：是否缺 hover、active、focus、loading、empty、error、成功反馈与当前导航定位。
5. 组件模式：是否卡片泛滥、FAQ 必手风琴、价格表必三塔、主次按钮永远一实一空、标签与徽标过度药丸化。
6. 文案与内容：是否满是 AI 套话、假公司名、假整数字、假头像、标题全 Title Case、成功提示过躁。
7. 代码与可访问性：是否缺语义标签、alt、meta、验证、skip link，或留下死代码、幻觉 import、任意 z-index。
8. 遗留残片：是否还留旧容器、旧 token、旧变体、旧状态样式、旧截图、旧说明与兼容旧视觉的临时补丁。

## 重设计修复次序

若问题很多，默认按以下顺序收：

1. 字体与字阶
2. 色盘与中性色统一
3. hover / active / focus 等交互反馈
4. 布局、容器与间距
5. 替换最模板化的组件模式
6. 删旧容器、旧 token、旧变体与旧说明，收掉新旧双轨
7. 补 loading / empty / error / success
8. 最后抛光字距、节奏与细部材质

## 视觉骨架顺序

1. 先定首屏任务：第一眼要看懂什么，第一步要做什么。
2. 再定色彩承诺：克制单强调、明确主色承担面，还是少量多角色配色。工具页默认先克制，品牌页可更果断。
3. 再定字体系：显示字、正文、标签、数字/等宽信息各担何职；正文行长常控在 `65-75ch`。
4. 再定容器逻辑：哪些内容真需要边界，哪些只靠对齐、留白、分隔即可成立。
5. 末定记忆点：同屏通常只保留一个主焦点与一处可记住的细节，不到处抢戏。

新建题可放大“记忆点”；重设计题先清旧病，再决定是否补新的记忆点，不可在旧骨架未正时硬塞亮点。

## 色彩与材质

1. 先收中性色温：背景、表面、边界、弱文本与强文本不得冷暖灰乱混。
2. 主色只承担一个核心角色：品牌、行动或状态择其一；语义色另行定义，不与品牌色抢职责。
3. 高密工具页少用大面积渐变、模糊、发光与厚阴影；需要质感时优先靠比例、对比、边界与微交互。
4. 品牌页可放大材质和摄影/位图资产，但文字、主 CTA 与下一段内容仍要清楚。
5. 暗色模式不是反色：须重定背景层级、边界亮度、语义色饱和、焦点可见性与图表颜色。

## 首屏规则

1. 首屏只保一个主焦点与一个主 CTA；其余皆退为次级。
2. 大标题优先 `1-3` 行；若文案过长，先缩句，不先堆行。
3. 高频工具页不套营销 hero、夸张指标墙或炫技首屏；先让状态与主路径可见。
4. 小笔电也要成立：缩到较窄桌面宽度，首屏仍看得见主信息、主操作与可信反馈。
5. 文字不得压住下一段或被容器裁切；按钮、标签、表格头和导航项先按最长文案验一次。

若是现有项目重设计，首屏先修主路径、主层级与主状态，再谈风格放大。

## 排版与布局守则

1. 先用字重、字阶、行距、留白定主次，再谈材质与特效。
2. 间距要有节奏，不要每块同 padding、同卡宽、同行高。
3. 居中不是默认。构图张力大于 `4` 时，优先考虑偏移、分栏、错位或不均匀留白。
4. 盒子只在真有分组、承载或状态边界意义时才出现；无理由不用大包裹容器。
5. 同类区块避免一页反复“三等宽卡片 + 图标 + 标题 + 说明”模板。
6. 工具页优先用工具栏、分隔线、列表、表格、侧栏、抽屉和紧凑状态区组织信息；卡片只用于可独立比较或承载状态的对象。
7. 固定格式控件要有稳定尺寸或响应约束；hover、加载文案、图标替换和数字变化不得推挤布局。

重设计时，优先删多余容器、并卡片层级、减药丸标签，并顺手清理依附其上的旧样式与旧说明，再补新的布局动作。

## 反模板味禁项

1. 紫蓝 AI 渐变、外发光、装饰性玻璃态默认铺满全页。
2. 彩色竖条提示卡、渐变文字、默认居中暗黑 hero、伪科技霓虹边。
3. 卡套卡、区块外再包大圆角大容器、页面像被一层层盒子困住。
4. 假复杂度：小药丸、状态签、系统编号、伪运行标签、无意义微文案堆满首屏。
5. 多主色、多主 CTA、多抢眼模块并列。
6. 只画理想态，不补加载、空、错、禁用、成功、回退与窄屏。

## 图像先行口径

若宿主支持图像生成，且任务主矛盾在视觉方向探索、官网改版、品牌页、创意首页或重设计，可先走图像先行：

1. 先按 section 生成参考，而非把整站压成一张小图。
2. 一张图看不清文字、按钮、间距或组件，就重生该 section，不裁旧大图。
3. 看图时至少抽：文字、字阶关系、留白、按钮形态、颜色角色、容器逻辑、记忆点。
4. 图只是定视觉，不是替代规格；仍要回写成可实现的设计草案。

若当前宿主未暴露图像生成能力，则改用“分 section 文字构图稿”：逐段写清主次、文字量级、组件骨架、图文比、节奏与禁项，再交实现。

若是现有项目重设计，图像先行只可作方向探针，不得绕开对现有代码、组件与状态的审计。

## 最小交付

命中本文件时，最终草案至少补齐：

- 页面体裁：品牌页 / 工具页 / 混合页
- 任务性质：新建前端 / 现有项目重设计
- 设计宣言：问题 / 气质极点 / 约束 / 记忆点
- 三拨盘：构图张力 / 动效分量 / 信息密度
- 首屏任务与唯一主操作
- 色彩承诺与字体系分工
- 容器、间距、布局与记忆点口径
- 状态与交互触感清单
- 反模板味禁项
- 若为重设计：现状病灶、遗留清理清单、薄改顺序与不可重写边界
- 状态完整性与响应式降级口径
- 是否需要图像先行，以及若不可用时的文字替代法

## 快检

出稿前至少过一遍：

1. 去掉颜色与特效后，主次是否仍成立。
2. 缩到小笔电或窄栏后，首屏是否仍清楚。
3. 是否还残留居中大标题配三卡片、卡套卡或伪技术小标签堆叠。
4. 是否只有一个主焦点，而不是多个强调位争抢。
5. 这页更像真实产品界面，而不是训练集里常见的模板站。
6. 最长文案、空/错/加载态、窄屏和高密度数据下是否仍不挤、不乱、不跳。
