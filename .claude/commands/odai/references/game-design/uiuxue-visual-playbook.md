# Game UI UX UE Visual Playbook

## 何时读取

游戏 HUD、菜单、背包、商城、编队、养成页、战斗内外界面；信息层级、操作负担、可读性、反馈节奏；触屏/手柄/键鼠输入；引导、弱网、低端机、状态说明。若要更高级/更有质感/反页游后台/反廉价，同读 `odai/references/game-design/aesthetic-benchmark.md`。

## 最小顺序

1. 场景：战斗内、局外养成、社交、商城、引导、活动页。
2. 目标：做决定、看状态、承接反馈、完成高频操作。
3. 参考系：同品类成熟作品借什么层级、节奏、材质、反馈；本题材绝不能像什么。
4. 层级：第一眼必读、第二眼可读、延后暴露、危险提示。
5. 质感：布局、间距、字体、图标、色彩、动效如何服从题材，避免网页/后台感。
6. 验证：误触、信息负担、反馈时机、设备适配、最低视觉质量。

## 最短查单

高/低压场景与可停留时长；主/次/危险操作分层；数值/资源/冷却/状态/引导是否抢焦点；成功/失败/锁定/冷却/异常/回退是否一眼可辨；是否落入网页后台/页游/电商皮；输入变化后是否顺手；弱网/掉帧/低端机/小屏下核心信息是否可读可操作。

## 高审美守则

先像游戏，再像产品；层级先于花活；质感靠字阶、留白、图标语法、材质分层、反馈节奏，不靠重描边/镀铬/高饱和霓虹/粒子雨。高压场景宁减不加；低压场景可丰但主操作、资源变化、稀有度须一眼分明。同套系统同套语法：按钮、资源、标签、稀有度、警示、禁用、锁定、冷却、可领取跨页一致。动效帮判断，不遮按钮、不吃帧、不盖判定/收益。UI 文案、数值、奖励名按本地化最长文本、小屏、刘海安全区、手柄焦点、弱网态验一次。

## 禁项

网页后台、SaaS 卡片、大表格、营销 hero 套进游戏主界面；金属框、发光描边、厚投影、粒子、爆闪冒充高级；商城/抽卡/活动过度抢主流程；只画理想态不补锁定/冷却/失败/断网/回退/降级；为了爽感让按钮、跳过、确认、收益、危险提示看不清。

## 最小交付

HUD/菜单草案：信息层级、区域分布、主次操作、状态反馈。流程说明：入口、切换、返回、异常、成功反馈。优化清单：输入负担、误触、可读性、质感、反廉价、动效、设备适配。UI 验收口径：关键状态、关键路径、可读性、操作完成。实装复核口径：最长文本、小屏安全区、焦点路径、弱网/掉帧降级、确认/跳过/返回可达。
