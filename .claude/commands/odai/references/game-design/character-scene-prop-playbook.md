# Character Scene Prop Playbook

## 何时读取

命中以下任一主题时，读本文件：

- 角色、怪物、NPC、Boss、载具、宠物或道具视觉方向
- 场景、地表、建筑、机关、关卡装置或世界观视觉规则
- 阵营识别、形体语言、材质层级和资产复用
- 游戏内长期运营下的角色 / 场景视觉统一性

## 最小顺序

1. 先定题材、目标玩家和观看距离：玩家主要在什么视距、什么设备和什么频次下看这些资产。
2. 再定识别锚点：阵营、职业、稀有度、危险性或功能性靠什么被一眼识别。
3. 再定层级：主角、敌人、NPC、场景、道具之间谁该抢焦点，谁只做支撑。
4. 末补验收：识别性、一致性、复用性与产能边界是否成立。

## 角色视觉短框架

至少补齐：题材定位、阵营差异、形体语言；色彩系统、材质倾向、细节密度；主角/敌人/NPC 在剪影和远视距下的区分；稀有度/职业/势力如何建立可读层级。

## 场景与道具短框架

至少补齐：世界规则、区域主题、功能区分；建筑/机关/装置/可互动物的功能可读性；场景层级、前中后景、探索引导；道具/载具/资源点与战斗判定信息是否互扰。

## 资产与产能

至少补齐：哪些资产必须高精度，哪些可模板化/模块化；角色/场景/道具可复用规则与部件；长期运营或多版本更新最易失控的视觉债。

## 最小交付

按主矛盾选：

- 角色视觉方向：题材、形体、色彩、识别性、一致性和风险
- 场景 / 道具视觉方向：世界规则、层级、功能可读性、复用与风险
- 角色 / 场景统一规则：阵营、材质、视距、细节密度和资产约束
