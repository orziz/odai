# Game UI UX UE Visual Playbook

## 何时读取

由 `game-design`「何时读取参考文件」触发；本文件覆盖游戏 HUD / 菜单 / 背包等界面的信息层级、操作负担、可读性与反馈节奏。

## 最小执行顺序

1. 先定场景：战斗内、局外养成、社交、商城、引导还是活动页。
2. 再定目标：本界面是让玩家做决定、看状态、承接反馈，还是完成高频操作。
3. 再定层级：哪些信息必须第一眼看到，哪些可以延后暴露。
4. 最后补验证：误触风险、信息负担、可读性、反馈时机和设备适配是否成立。

## 最短检查清单

至少补齐：

- 当前界面是高压场景还是低压场景，允许玩家停留多久
- 主操作、次操作、危险操作是否清楚分层
- 数值、资源、冷却、状态、引导提示是否抢夺同一视觉焦点
- 反馈是否足够快，成功、失败、锁定、冷却和异常是否能一眼辨认
- 输入方式变化后，点击、拖拽、长按、摇杆或快捷键是否仍然顺手
- 弱网、掉帧、低端机或小屏环境下，核心信息是否仍能读懂和完成操作

## 按界面类型判断

- 战斗 HUD：先保读招、危险、血量 / 资源、技能冷却、目标锁定和收益反馈；装饰层不得压过判定信息。
- 养成 / 背包：先保筛选、比较、强化前后差异、资源消耗、不可逆提示和返回路径；稀有度表现不得盖住可决策信息。
- 商城 / 抽卡 / 活动：先保价格、概率 / 规则入口、剩余时间、奖励池、领取状态和误购防线；商业包装不得反压主玩法。
- 编队 / 构筑：先保角色定位、阵容关系、冲突提示、替换成本、保存状态和失败原因；不要只展示单卡强度。
- 新手引导：每步只教一个决策点；遮罩、箭头、奖励弹窗不得抢走正在学习的主操作。

## 状态矩阵

至少覆盖正常、空、锁定、冷却、资源不足、可领取、已领取、失败、断线 / 重连、跳过 / 返回。高频界面还要覆盖批量操作、撤销、确认二次弹层和长时间停留后的状态刷新。

## 红线

主按钮被特效 / 弹窗遮挡；危险信息和奖励信息共用同一焦点；触屏热区小于同平台常规操作预期；手柄焦点丢失；关闭弹窗后玩家不知道下一步；静态截图好看但压力态读不出判定。

## 常见交付

按问题主矛盾选择最小交付：

- HUD / 菜单草案：信息层级、区域分布、主次操作、状态反馈
- 界面流程说明：入口、切换、返回、异常态和成功反馈
- 体验优化清单：输入负担、误触风险、可读性、动效节奏、设备适配
- UI 验收口径：关键状态、关键路径、可读性和操作完成标准
