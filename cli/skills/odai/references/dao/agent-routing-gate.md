# 模型 / Agent 前置路由短门

## 何时读取

需求、边界与验收已足以选择执行档位，且即将进入深分析、大范围读取、重执行或正式冻结时读取。本文件只决定是否需要展开完整 `references/dao/agent-governance.md`，不执行模型路由或 agent 下放。

## 短判

命中以下任一条，读取完整 `references/dao/agent-governance.md`：

- 用户显式要求模型切换、指定 / 比较模型、按成本路由、并行或 agent 下放，或题面已明确冻结并选择 agent 下放。
- 当前执行档换过正交方向后仍在同一判别性验收失败，或已有上下文、模态、工具 / 推理能力缺口的直接证据，需要判断是否上调能力而不是继续盲试。
- 任务已稳定、可清晰切分，且长上下文扫读、并行证据、候选实现或重执行的收益明显高于冷启动、交接与复核成本。
- 待冻结结论高代价或难回退；此时先读 `references/dao/decision-challenge.md`，只有裁决需要真实调用时才展开完整 agent 治理。

都未命中时，留在主流程，不读完整 agent 手册，不因路由判断延迟推进。宿主未真实暴露所需能力时，不得声称已切模型、已并行或已调用 agent。
