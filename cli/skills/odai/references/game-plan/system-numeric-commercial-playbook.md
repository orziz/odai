# System Numeric Commercial Playbook

## 何时读取

任务主矛盾落在系统结构、循环、成长、功能耦合、数值框架、平衡、资源 / 经济循环、货币分层、商业化、付费结构或公平性边界时读取。

## 最小顺序

定核心体验、目标玩家、胜负 / 成长目标 → 拆系统循环、资源流向、数值区间、商业边界 → 标风险（平衡破坏、资源断档、成长塌陷、付费压迫、公平性争议）→ 选择交付（系统方案、数值框架、经济说明或商业化方案）。

## 检查点

系统策划：玩家局内外反复做什么；胜负 / 完成条件是什么；系统之间如何喂数或限制；成长结构如何制造长期目标；哪些地方会跳过主循环形成破局解。不要只写功能列表，要写功能与循环的关系。

数值与经济：关键资源、来源、去向；角色 / 装备 / 关卡 / 对局的关键数值轴；成长曲线、难度曲线如何随时长或版本变化；正常、重度、付费玩家的产消比；哪些极值会打穿平衡或内容消耗速度。不要只给单点参数，要说明为什么这样定、后续如何调。

商业化：商业目标是首购、ARPPU、留存带动、赛季转化还是版本回收；付费点、货币体系、价值包装如何挂在循环和活动节奏里；公平性、社交压力、强制付费感、付费前置卡点边界；版本更新时商业结构如何迭代而不压垮原系统。不要把商城界面、商品文案或单次促销当商业模型本体。

## 系统关系图口径

至少说明：核心循环入口、局内目标、局外成长、资源来源、资源消耗、奖励回流、失败补偿、社交 / 商业插入点。每个新增系统都要回答它增强哪个循环、消耗什么资源、产出什么长期目标、会不会绕过主玩法。

## 数值骨架

- 定轴：伤害 / 防御 / 生命 / 时间 / 掉落 / 体力 / 货币等关键轴分别服务什么决策。
- 定曲线：早期教学、中期构筑、后期追求的成长斜率不同；不要一条线拉到底。
- 定极值：最高爆发、最低保底、最快毕业、最慢追赶、最大囤积和最坏抽样都要有边界。
- 定调参点：哪些参数可运营调整，哪些会破坏规则可信度，哪些必须发版。

## 经济与商业红线

核心货币来源失控、消耗口不足、付费直接跳过核心玩法、免费玩家无追赶路径、活动奖励打穿常驻经济、首购设计破坏后续价值锚点、概率 / 保底 / 限时规则无法被玩家理解。

## 最小交付

- 系统方案：目标、循环、系统关系、成长结构、风险。
- 数值框架：关键公式、区间、曲线、调参点、验证口径。
- 经济说明：资源层级、产消链路、风险点、版本影响。
- 商业化方案：付费结构、价值包装、边界、节奏与风险。
