# 统一术语基线

## 适用范围

- 本文件适用于 `odai` 入口、内部模块、support files、模板、主文件草案、结果总结与同步前校验。
- 只要涉及问题整理、结构化提问、证据账本、工作草案、主文件字段或路径命名，就统一沿用本文件，不再临时发明近义口径。

## 核心字段

1. 当前理解：当前这一轮已经整理出的整体判断，可继续被修正。
2. 当前理解的用户诉求：用户表面明确提出要做的事、要看的问题或要拿到的交付物。
3. 当前已确认的用户真实想法与在意点：只写已经被用户明确确认，或已被项目事实直接支撑的真实意图、顾虑、偏好与不可接受结果；没确认的内容一律移到未确认点。
4. 已知事实：用户原话、现有材料、仓库内容或上下文里可以直接看到，但尚未额外验证的事实性信息。
5. 已验证事实：已通过代码、文档、日志、测试、截图、页面观察或低风险验证确认过的事实。
6. 未确认点：任何仍拿不准、仍可能改写理解、路径、边界、验收、优先级或风险判断的地方。
7. 冲突点：用户说法、仓库事实、现有代码、现有文档、测试结果或其他证据之间互相打架的地方。
8. 必须确认的问题：由未确认点和冲突点收束出来、必须一次性向用户问清的问题组。
9. 待验证项：可以先由模型用仓库、材料或低风险验证自行补齐的项。
10. 待确认项：必须由用户拍板、授权或明确表态后才能继续的项。
11. 风险项：即使当前不阻塞，也可能影响质量、成本、边界、稳定性或长期维护的项。
12. 已确认路径 / 待确认路径：已经明确可以执行的路线，与仍需用户确认后才能继续的路线问题。

## 结果总结补充字段

1. 实际命中的内部模块 / 内部技能：本轮真实读取、展开、继承或实际命中的内部模块、内部技能。
2. 实际调用的 agent：本轮真实参与的主流程 agent 或子 agent；当前环境未暴露标识时写“当前环境未暴露”。
3. 实际调用的模型：本轮真实参与的模型；当前环境未暴露标识时写“当前环境未暴露”。

## 使用规则

- 提问前先给：已知事实、已验证事实、未确认点、冲突点、必须确认的问题。
- 任何未被用户明确确认、也未被项目事实直接验证的内容，都不能写进“已确认”类字段。
- 不得把模型自拟理解、补全推断、预设优选方向、未经确认的继续方向或默认答案写成事实或既定路径。
- 若用户自己已经给出多个说法、多个选项或多个方向，可以按其原话归并后请他确认；不要额外补出模型自拟选项。
- 只有在用户明确授权你代拟草案时，才允许给出“待确认草案”或“待确认路径”；未确认前不得当成推进依据。
- 结果总结若回报模块、agent 或模型，只能写真实命中与真实调用；当前环境未暴露对应标识时，必须明确写“当前环境未暴露”，不得补写、猜写或包装成已调用。

## 最小对外结构

若当前任务需要短输出，优先压成下面这组字段：

1. 当前理解
2. 当前理解的用户诉求
3. 当前已确认的用户真实想法与在意点
4. 已知事实 / 已验证事实
5. 未确认点 / 冲突点
6. 必须确认的问题
7. 已确认路径 / 待确认路径
8. 下一步
