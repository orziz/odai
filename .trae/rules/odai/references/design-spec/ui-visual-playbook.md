# UI Visual Playbook

## 何时读取

命中以下任一主题时，读本文件：

- 网站首页、品牌页、工作台、控制台、仪表盘、设置页、表单流、编辑器壳层
- 用户明要更高级、更像成品、更不像模板站或通用 SaaS
- 题目主矛盾不止在“美不美”，而在页面骨架、构图节奏、排版气质与反俗套
- 现有界面显居中大标题配三卡片、卡套卡、强调过多、伪技术小标签过密、首屏拥堵

若同时命中高审美提质，再同读 `odai/references/design-spec/aesthetic-benchmark.md`。

## 先判页面体裁

先断当前页面属于哪一类，再谈风格：

1. 品牌页：设计本身即卖点，可放大构图、留白、记忆点与情绪张力。
2. 工具页：设计服务任务完成，优先扫读、状态、反馈、连续操作与低认知负担。
3. 混合页：先断首要任务；若用户进来先做事，按工具页；先进来先被说服，按品牌页。

未先定体裁，极易把营销页语言误套进工具页，或把工具页做成无个性的中性壳。

## 三拨盘

出草案前，至少先给三项拨盘口径：

1. 构图张力 `1-10`
   - `1-3`：对称、稳、保守，适合设置页、后台基础页。
   - `4-7`：允许偏移、错位、大小对比与非均匀留白，适合大多数高质量产品页与工作台。
   - `8-10`：强非对称、显著负空间、章节节奏更戏剧化，只在品牌页或低频展示面慎用。
2. 动效分量 `1-10`
   - `1-3`：仅状态反馈、轻 hover、轻位移。
   - `4-7`：允许阶段入场、顺序揭示、空间切换与少量连续动效。
   - `8-10`：滚动编排、叙事转场、复杂互动；高频工具页默认不取高值。
3. 信息密度 `1-10`
   - `1-3`：疏朗、展陈感强，适合品牌与叙事页。
   - `4-7`：日常产品默认区间，清楚且不挤。
   - `8-10`：高密工具、监控、交易、运营台；此时优先行分组、分隔线与字阶，不靠大卡片装箱。

三拨盘先于细节风格。拨盘不定，后续排版、容器、动效与状态会互相打架。

## 视觉骨架顺序

1. 先定首屏任务：第一眼要看懂什么，第一步要做什么。
2. 再定色彩承诺：克制单强调、明确主色承担面，还是少量多角色配色。工具页默认先克制，品牌页可更果断。
3. 再定字体系：显示字、正文、标签、数字/等宽信息各担何职；正文行长常控在 `65-75ch`。
4. 再定容器逻辑：哪些内容真需要边界，哪些只靠对齐、留白、分隔即可成立。
5. 末定记忆点：同屏通常只保留一个主焦点与一处可记住的细节，不到处抢戏。

## 首屏规则

1. 首屏只保一个主焦点与一个主 CTA；其余皆退为次级。
2. 大标题优先 `1-3` 行；若文案过长，先缩句，不先堆行。
3. 高频工具页不套营销 hero、夸张指标墙或炫技首屏；先让状态与主路径可见。
4. 小笔电也要成立：缩到较窄桌面宽度，首屏仍看得见主信息、主操作与可信反馈。

## 排版与布局守则

1. 先用字重、字阶、行距、留白定主次，再谈材质与特效。
2. 间距要有节奏，不要每块同 padding、同卡宽、同行高。
3. 居中不是默认。构图张力大于 `4` 时，优先考虑偏移、分栏、错位或不均匀留白。
4. 盒子只在真有分组、承载或状态边界意义时才出现；无理由不用大包裹容器。
5. 同类区块避免一页反复“三等宽卡片 + 图标 + 标题 + 说明”模板。

## 反模板味禁项

1. 紫蓝 AI 渐变、外发光、装饰性玻璃态默认铺满全页。
2. 彩色竖条提示卡、渐变文字、默认居中暗黑 hero、伪科技霓虹边。
3. 卡套卡、区块外再包大圆角大容器、页面像被一层层盒子困住。
4. 假复杂度：小药丸、状态签、系统编号、伪运行标签、无意义微文案堆满首屏。
5. 多主色、多主 CTA、多抢眼模块并列。
6. 只画理想态，不补加载、空、错、禁用、成功、回退与窄屏。

## 图像先行口径

若宿主支持图像生成，且任务主矛盾在视觉方向探索、官网改版、品牌页、创意首页或重设计，可先走图像先行：

1. 先按 section 生成参考，而非把整站压成一张小图。
2. 一张图看不清文字、按钮、间距或组件，就重生该 section，不裁旧大图。
3. 看图时至少抽：文字、字阶关系、留白、按钮形态、颜色角色、容器逻辑、记忆点。
4. 图只是定视觉，不是替代规格；仍要回写成可实现的设计草案。

若当前宿主未暴露图像生成能力，则改用“分 section 文字构图稿”：逐段写清主次、文字量级、组件骨架、图文比、节奏与禁项，再交实现。

## 最小交付

命中本文件时，最终草案至少补齐：

- 页面体裁：品牌页 / 工具页 / 混合页
- 三拨盘：构图张力 / 动效分量 / 信息密度
- 首屏任务与唯一主操作
- 色彩承诺与字体系分工
- 容器、间距、布局与记忆点口径
- 反模板味禁项
- 状态完整性与响应式降级口径
- 是否需要图像先行，以及若不可用时的文字替代法

## 快检

出稿前至少过一遍：

1. 去掉颜色与特效后，主次是否仍成立。
2. 缩到小笔电或窄栏后，首屏是否仍清楚。
3. 是否还残留居中大标题配三卡片、卡套卡或伪技术小标签堆叠。
4. 是否只有一个主焦点，而不是多个强调位争抢。
5. 这页更像真实产品界面，而不是训练集里常见的模板站。
