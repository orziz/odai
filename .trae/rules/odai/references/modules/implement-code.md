## 模式

1. 默认直接实现：边界清、风险低、测试先行收益不明显时，直接小步改并验证。
2. 优先 TDD：用户明要测先行；bug 可稳复现；核心规则/算法/状态机；高风险重构；回归成本高。
3. 真正 TDD 须目标行为/验收已稳且能自动化表达失败；行为漂浮则回上游补规格。
4. 一次性脚本、低风险样式/文案、纯搬运、无测试接缝时，不强推 TDD。

## 实现规则

1. 先读相关代码、文档、测试、近似模式；可由代码/日志/文档直补的事实不反问。
2. 路未定或无法确认合用户真意，停在补证据/补确认，不把推断写进实现。
3. 优先局部改、局部验、必要抽象；能改旧路不开新路。代码求短链路、低嵌套、少状态、少依赖、少间接层。
4. 先做减法：删旧码、并分支、减层级/参数/状态/依赖；多轮变更先扫补丁叠补丁、旁路、临时兼容层、假通用层。
5. 抽象/组件化须有真实重复、稳定边界、维护净收益；不为预留未来造 helper/manager/wrapper/base/adapter/hook。
6. 已替历史遗留者，同轮同界删净死代码、废分支、废 props/状态/样式/测试、过期文档；跨文件旧债只记风险，不借机全清。
7. 涉 UI，先读页面、组件库、样式系统、design token、设计文档、邻页模式。审美升级/重设计/基线冲突时，升 `design-spec` 并读 `aesthetic-benchmark`、`ui-visual-playbook`；不自造规则、不重写框架、不另起平行体系。
8. UI 实装同查组件复用、一致性、normal/hover/focus/active/selected/disabled/loading/empty/error/success/回退、响应式、文本溢出、键盘可达；不只修静态理想态。
9. 同查功能正确性、边界、错误流、回退路径、监听/订阅/定时器/缓存/连接/引用释放。
10. 性能只优化真热路径、真瓶颈、真重复；无证据不引入缓存、异步化、泛型层、配置层或过早优化。
11. 小修不借机换架构/范式/状态模型/项目级约定；确需升架构，先回上游说明收益与代价。
12. 改后跑相关测试/build/lint/format/低风险验证；跑不了直说。UI 缺自动化时，至少说明已查状态、最长文案/窄屏/高密度、遗留清理与未验证项；可做浏览器观察时验非空、无遮挡、无溢出、关键反馈。

## 冲突与越线

优先级：用户明确要求 > 项目现有代码/目录/工具链 > 项目文档 > 生态主流 > 本模块默认项。

破坏/不可逆/大范围重构/项目级依赖/根配置/发布部署/外部访问/费用/数据安全，先问授权、影响面、回滚、验收。默认不做全仓格式化、根级 lint/build/CI 改写。未跑即未跑；不得编造验证结果。
