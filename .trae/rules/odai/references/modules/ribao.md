司成果说明。把当天工作写成同事、评审、协作者一看即懂、可直发之白话中文；不用文言，不装腔，不套话。

## 输入来源（优先级）

1. 用户明确描述的工作内容、结果和问题
2. 默认范围内的 git 记录与当前改动
3. 默认范围内有修改痕迹的文件内容
4. 对话上下文补充信息

以用户明确描述为准；git 信息与文件内容只用于补充和校验。

## 共通主轴

日报、commit message、PR message 皆说清三事：
- 做了什么
- 为什么做
- 结果或价值是什么

先提共通表达，再按用途轻改；不拆三套互不相干话术。

## 默认范围

未特别说明时，默认按“今天 00:00 到现在”取材：

- 今天的 git commits，用于归纳已提交产出
- 当前 staged / unstaged 改动，用于补充尚未提交的工作
- 今天有修改痕迹的文件内容，用于辅助判断真实工作内容

文件改动只作旁证；若与用户描述或 git 冲突，以用户描述和 git 为准。

## 默认输出

未特别说明时，默认输出简洁版：

- 每条含“动作 + 目的或结果”
- 可直用于日报，亦可轻改为 commit 或 PR 文案
- 不写流水账，不机械列文件名
- 语气自然，像真实工作汇报

若用户要求详细版，再补背景、处理方式和影响。

## 无有效输入时

若无可用工作内容与 git 信息，给一条克制幽默的“摸鱼版”内容；仍保持汇报感，不写段子。

## 风格与适配

1. 默认使用中文；若用户指定其他语言，再跟随用户。
2. 不加“下面是你的日报”这类引导语。
3. 不用代码块包正文。
4. 不编造未被用户或 git 信息支持的事实。
5. 日报突出当天产出与结果；commit message 凝练本次提交；PR message 在简洁版上略展背景与改动点。
6. 未指定用途，先给通用简洁版；要多版，再分别给日报/commit/PR 版。
