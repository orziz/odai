# Game UI UX UE Visual Playbook

## 何时读取

命中以下任一主题时，读本文件：

- 游戏 HUD、菜单、背包、商城、编队、养成页或战斗内外界面
- 游戏中的信息层级、操作负担、可读性和反馈节奏
- 触屏、手柄、键鼠等不同输入方式下的交互体验
- 游戏场景里的引导、弱网反馈、低端机可用性和状态说明

用户明要更高级、更有质感、更不像页游/网页后台，或现有游戏 UI 显廉价、拼贴、商城感过重时，先同读 `odai/references/game-design/aesthetic-benchmark.md`。

## 最小顺序

1. 先定场景：战斗内、局外养成、社交、商城、引导还是活动页。
2. 再定目标：本界面是让玩家做决定、看状态、承接反馈，还是完成高频操作。
3. 再定参考系：同品类成熟作品借什么层级、节奏、材质与反馈；本题材绝不能像什么。
4. 再定层级：哪些信息必须第一眼看到，哪些可以延后暴露。
5. 再定质感：布局、间距、字体、图标、色彩、动效与题材气质如何统一，避免通用网页/后台感。
6. 末补验证：误触风险、信息负担、可读性、反馈时机、设备适配与最低视觉质量是否成立。

## 最短查单

至少补齐：高/低压场景与可停留时长；主/次/危险操作分层；数值/资源/冷却/状态/引导是否抢焦点；成功/失败/锁定/冷却/异常/回退反馈是否一眼可辨；布局/间距/字体/图标/色彩/动效是否有游戏质感；是否仍落入网页后台 / 页游 / 电商商城皮相；输入变化后仍否顺手；弱网/掉帧/低端机/小屏下核心信息是否可读可操作。

## 高审美守则

1. 先像游戏，再像产品。先立题材、压力、节奏和操作感，再立装饰。
2. 层级先于花活。主目标、敌我状态、资源与危险提示必须先于材质与特效被读到。
3. 质感靠秩序，不靠堆料。优先用字阶、留白、图标语法、材质分层和反馈节奏建立高级感，不靠重描边、镀铬、高饱和霓虹和粒子雨硬撑。
4. 高压场景宁减不加。战斗、追击、限时决策时，能晚露出的信息就别早露。
5. 低压场景也不散。养成、背包、商城、编队允许更丰，但仍要让主操作、资源变化和稀有度一眼分明。
6. 同套系统要同套语法。按钮、资源、标签、稀有度、警示、禁用、锁定、冷却、可领取都要跨页一致。
7. 动效要帮玩家判断，不抢玩家判断。爽感可有，但不能遮按钮、吃帧或盖住判定与收益。

## 禁项

1. 把网页后台、SaaS 卡片、大表格、营销 hero 直接套进游戏主界面。
2. 只靠金属框、发光描边、厚投影、粒子和爆闪冒充高级。
3. 商城、抽卡、活动页过度抢焦点，反压主流程与日常高频入口。
4. 只画静态理想态，不补锁定、冷却、失败、断网、回退和性能降级。
5. 为了“爽”让按钮、跳过、确认、收益结算或危险提示看不清。

## 最小交付

按主矛盾选：

- HUD / 菜单草案：信息层级、区域分布、主次操作、状态反馈
- 界面流程说明：入口、切换、返回、异常态和成功反馈
- 体验优化清单：输入负担、误触风险、可读性、视觉质感、反廉价禁项、动效节奏、设备适配
- UI 验收口径：关键状态、关键路径、可读性和操作完成标准
