# 并行合议触发短判

## 何时先读本文件

- 需要判断是否开启增强模式、多 agent / 多模型合议、当前模型三席模拟复核或子 agent 辅助复核。
- 需要判断冻结方案后是否值得做独立复查。
- 需要解释为什么本轮不启用并行复核。

## 默认行为

**短判结果为“原流程”时，不读完整 playbook**——这是绝大多数任务的正常路径，不需要额外说明或证明。完整 playbook（440 行）仅在以下情况读取：
1. 用户显式要求增强模式、多模型合议或并行复核。
2. 短判裁决为“完整多模型合议”且能力现状确认可用。
3. 冻结方案后需要独立复查且短判确认值得。

短判结果为“当前模型三席模拟复核”或“子 agent 辅助复核”时，可只按需读取完整 playbook 中对应章节，不全文展开。

## 五步短判

1. 阶段是否合法：只在问题理解、方案讨论、决策收束、冻结方案后的独立复查阶段考虑并行；已冻结方案下的实现、验证、补证或文档执行分工按原流程模式处理。
2. 复杂度是否值得：命中两项以上中 / 大复杂信号，或单独命中高代价冻结点、误判回滚成本高，才默认进入增强尝试链。
3. 是否有排除信号：单文件小改、低风险修辞、主路单一、成熟模式下局部实现、回滚成本低时，优先原流程。
4. 能力是否真实可用：只有当前环境真实暴露子 agent / 并行能力、模型派位能力、同版 skill 下发能力与权限 / 配额可用性，才可进入对应档位；不能用模型自述、旧日志、角色名或别名伪造多模型。
5. 裁决落哪一档：完整多模型合议 -> 当前模型三席模拟复核 -> 子 agent 辅助复核 -> 原流程。每降一档都要重判收益，收益不足就回原流程。

## 中 / 大复杂信号

- 同时存在两条以上合理主路，且取舍会改变范围、边界、验收、成本或风险接受度。
- 任务跨两个以上专项模块，且模块间存在真实牵引。
- 当前裁决属于方向定案、规格冻结、架构主路、广范围审查结论等高代价冻结点。
- 原流程已收束一轮后仍有会改写路线、边界、验收或不可接受结果的剩余分歧。
- 不并行复核的误判回滚成本显著高于一次复核成本。
