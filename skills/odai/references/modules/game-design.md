此模块司游戏视觉：先明视觉目标、目标玩家、现有约束与不可受之果，再收 UI/UX/UE、角色、场景、道具、宣发包装、品牌气质与特效演出为交接文。

此模块主收游戏视觉；题目若实为通用页面、交互或体验设计，则回 `design-spec`；本体若仍是系统、数值、经济、商业或关卡规划，则回 `game-plan`。

输入纵散，先敛为已知、未定、相左与必问；主动补看可读性、一致性、信息负担、风格漂移、资产复用与维护；未证者只列待定、待验或风险。

定位：只司游戏视觉说明、交接草案与方向整理，不入代码实现、资源制作或引擎配置。

先清视觉基线，再定执行；基线未稳，不下放执行。

用户明要更高级、更有质感、更不像廉价手游/网页后台，或现有游戏界面显拼贴、页游感、商城感过重时，先读 `references/game-design/aesthetic-benchmark.md` 定视觉标尺，再出草案。

## 执行要点

1. 先明所求/所想/玩家/视觉目标/不可受结果，再定界面/角色/场景/品牌/特效。
2. 十二层不混：视觉目标、玩家、平台与观看距、关键场景、UI/UX/UE、角色场景道具、品牌宣发、特效演出、视觉偏好、信息层级、约束、证据。
3. 输入散者，先敛已知/未定/相左/缺口，再确认；不令用户从零重讲。
4. UI/UX/UE 看 HUD、菜单、背包、商城、编队、养成、战斗内外、引导、输入负担、信息密度、可读性、设备适配、游戏感、题材融入与动效节奏。
5. 涉 UI/UX/UE 审美升级，先定参考系：同品类成熟作品只借层级、读招、资源布局、材质分寸与节奏，不抄整套壳、图标或皮相；再按平台输入、镜头距离与高低压场景重立骨架。
5. 角色/场景/道具看题材、形体、色彩、材质层级、阵营区分、识别性、资产复用。
6. 品牌宣发看卖点、受众预期、品牌气质、传播场景、活动包装、商店展示一致性。
7. 特效演出看战斗反馈、读招清晰、强弱节奏、镜头配合、性能压力、观看疲劳。
8. 补看抢焦点、通用后台感、廉价感、页游感、风格漂移、资产复用、宣发与游戏内割裂、特效遮判定、低端机降级、维护价；未证只列待定/待验/风险。
9. 游戏 UI 不套通用网页/后台皮；先按题材、镜头距离、操作压力与平台定层级、质感、动效、音画反馈和最低可读标准。不得以金属边框、重描边、霓虹高光、粒子堆叠或大面积伪 3D 装饰硬撑“高级感”。
10. 主矛盾属系统/数值/经济/商业/关卡本体者，先回 `game-plan`；属非游戏页面/交互/信息架构/体验说明者，回 `design-spec`。
11. 需落文，早起 Markdown 草案；本模块止于视觉说明、方向、验收与交接。

## 理解、交付与边界

1. 交付按主矛盾取一：视觉草案、UI/UX/UE 方案、角色 / 场景方向、品牌宣发方案、特效 / 演出方向、视觉验收口径、AI 任务单或规划 Markdown。
2. 产出视觉文档或用户要落盘者，可直写项目 Markdown；后续续同文，不开平行版。
3. 不做代码、配置、脚本、测试、资源制作或数据实施；若入实现，只交视觉目标、规则、验收与风险。

## 何时必须读取参考文件

以下内容不必一开始全文背诵，但遇到对应场景时必须展开：

- `references/game-design/coverage-matrix.md`
  涉完整游戏视觉设计判定，或 `game-design` / `design-spec` / `game-plan` 裁边界，即读。
- `references/game-design/aesthetic-benchmark.md`
  涉游戏 UI/UX/UE 审美升级、质感提纯、反廉价、对标成熟作品或现有界面显网页/后台感时，即读。
- `references/game-design/uiuxue-visual-playbook.md`
  涉 HUD、菜单、背包、商城、编队、养成页、战斗内外界面、信息层级、交互负担或体验反馈，即读。
- `references/game-design/character-scene-prop-playbook.md`
  涉角色、怪物、NPC、场景、道具、载具或世界观视觉方向，即读。
- `references/game-design/brand-packaging-playbook.md`
  涉品牌气质、KV、海报、商店图、活动包装、宣发物料或传播视觉，即读。
- `references/game-design/fx-cinematics-playbook.md`
  涉技能特效、镜头演出、转场包装、战斗反馈或性能降级边界，即读。

字段命名、问题分组或交接草案结构成主矛盾时，再补读 `references/dao/terminology-baseline.md`。

默认先按本文件主规则推进；命中上述场景，再展开对应 reference。
