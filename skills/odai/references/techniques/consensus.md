# 合议与多模型复核

仅在用户明确要求合议、多模型复核、增强压力测试，或高代价决定经一次独立挑战后仍需要重型复核且用户同意时读取。默认任务不加载。

## 组织

- 只使用宿主真实提供的模型和 agent；默认三席，可按问题与用户要求调整。无法满足点名模型或真实异构性时，说明实际能力与退化方式，不伪装。
- 各席接收相同的冻结题面、完整正反证据、范围、允许动作和停止条件；不要泄漏预期答案或让一席先看其他席结论。
- 分配不同关注面，例如目标 / 价值、技术 / 可行性、风险 / 反例，但每席都可推翻前提。
- 合议只提供证据和挑战，不替代用户对目标、授权、成本和不可接受结果的决定。

## 收束

主流程按证据质量、假设覆盖和实际影响综合，不按多数票。对每项关键分歧标明采纳、部分采纳或驳回及理由；新证据动摇题面时回到上游重新判断。

最终只报告真实参与的席位、模型、分工、关键分歧和结论。执行分工、权限与结果采纳按 `references/dao/coordination.md`。
