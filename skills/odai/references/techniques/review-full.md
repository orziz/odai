# 三省六部完整审查模式

## 何时读取

用户要求完整模式，或结论将作为合并、发布、安全、全仓 / 全目录准入门时读取。范围、严重度与通用纪律仍以 `references/capabilities/review-sslb.md` 为准。

## 输出模板

- 【中书省·审前研判】：审查范围、变更意图、主审文件、疑似未用文件筛查、主审方向、审查重点。
- 【尚书省·任务派发】：审查优先级（XX部 > XX部 > ...）与分工。
- 【XX部】：`🔴 严重｜file_path:line — 问题；原因；影响 → 建议`；`🟡 建议｜...`；无问题写 `🟢 本部职责范围内未发现问题`。
- 【门下省·终审】：总计、裁决（✅ 准予合并 / ⚠️ 修改后合并 / ❌ 驳回重写 / 🕯️ 留中待问）、必须修改、建议优化、可暂缓、留中待问、疑似未使用文件。
- 【锦衣卫·监察密报】：越权 / 遗漏 / 误判 / 定级失当 / 流程违规；证据不足写 🕯️ 留中待问；未发现写 ✅ 未发现违规。

## 裁决

裁决按风险和验收影响，不按 🟡 数量机械计数：

- 无 🔴，且没有会阻断当前验收、发布条件或已确认边界的 🟡 时，可准予合并。
- 问题明确可修且会影响当前准入时，裁决修改后合并。
- 方向性错误、架构性问题或多处严重缺陷时，驳回重写。
- 锦衣卫要求降级、撤销或留中待问时，门下省必须同步改写结论。
