# 正式与收敛审查

仅在任务明确要求正式完整报告、连续 N 轮 clean、全仓 / 全目录复查、安全 audit，或结论将作为合并、发布、安全准入门时读取。普通单轮审查不读；范围、严重度与通用纪律仍以 `references/capabilities/review.md` 为准。

## 连续审查与修复收敛

开始前冻结审查范围、证据基线、相关验证和 clean 判定。默认 clean 表示没有阻塞当前目标的 finding 且相关验证未失败；用户要求零 finding 时从用户。阻塞 finding 需要当前范围可触发的路径和实际影响，证据不足留为待确认或非阻塞观察；维护建议与真实缺陷分开。

每轮基于当前 checkout 和同一冻结范围重新检查。范围内修改、证据基线变化、阻塞 finding 或相关验证失败会使连续 clean 计数归零；明确无关的范围外变化不重置。紧凑记录：`clean round x/N｜结果｜范围｜关键证据`，不能复用修复前结论、旧线程自报或未完成扫描。

使用 agent 时按 `references/dao/leverage.md`：扫描者只读并回交 finding 与复现证据，主审负责去重、定级和计数。用户已授权修复时，先冻结经核证的完整清单，再按 `references/capabilities/delivery.md` 实施；修复后重跑相关验证并把 clean 计数归零，只有主审实际复验后才关闭 finding。

## 正式准入输出

- **审前基线**：范围、变更意图、证据基线、主审文件、排除项与重点。
- **Findings**：按严重度给定位、触发条件、影响、最小修复和复验；没有问题不凑数。
- **准入裁决**：准予 / 修改后准予 / 驳回 / 待问，并列必须修改、可暂缓和未覆盖项。
- **过程完整性**：说明遗漏、误判、定级争议、工具失败或范围变化；没有就明确未发现。

裁决按风险和验收影响，不按建议数量机械计数：无阻塞问题且发布条件与已确认边界成立时可准予；问题明确可修且影响准入时修改后准予；方向性或多处严重缺陷时驳回；证据、范围或责任人未定时待问。
