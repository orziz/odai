# UI 审美与交互复核清单

## 何时读取

页面丑/廉价/像模板/AI 味/缺高级感；交互别扭、状态不全、反馈不清、回退不顺；审查对象是页面、组件库、设计稿、设计说明、UI PR、前端实现或 design system 调整时读本文件。

## 审查顺序

1. 基线：design system、组件、token、视觉规则、审美标尺、状态矩阵是否存在。
2. 主次：三秒内能否看出页面唯一主角、首要信息、解释 / 证据、主操作、次级信息和危险操作。
3. 系统：栅格、间距、字阶、颜色角色、圆角、边框、阴影、动效是否像一套。
4. 状态：normal、hover、focus、active、selected、disabled、loading、empty、error、success、回退是否齐。
5. 压力态：最长文案、窄屏、小窗、宽屏 / 超宽屏、高密度、低性能、键盘路径是否可读可用。
6. 代价：感知性能、认知负担、响应式、可访问性、复用维护成本是否过界。

## 非补偿式准入门

关键维度分别裁决，不做总分平均：主任务 / 主操作可辨；空间在目标视口完整；内容与信任证据真实；排版与组件属于同一系统；关键路径、状态与响应式可用；焦点、对比度、命中面积等可访问性成立。任一关键维度出现阻断，整体不得判可验收，其他细节精致不能抵消。

界面内容若靠无来源的指标、客户 / 评价、奖项 / 认证、联系方式、法律标识、产品截图或运行数据建立可信度，按关键真实性失败处理；删除、明确占位或补可核来源后再验。

## 浏览器证据

- 浏览器能力可用时，以真实页面、真实 viewport 和核心交互为证据；截图数量、源码检查、构建、lint 或浏览器窗口尺寸不能替代真实 viewport 验收。
- 从候选矩阵选择最小有判别力集合：产品最小宽度或 `390x844`、相关中间断点或 `768x1024`、常规桌面 `1440x900`、宽屏 `1920x1080`、超宽 `2560x1080`。只有全幅、分栏、固定宽度、影院式构图或宽屏空间完整性存在风险时才要求超宽；断点 / 容器 / 导航 / fixed / sticky 变化须覆盖断点两侧和一个相关极端。
- 浏览器不可用时只报告静态风险并标视觉未验证，交付实际需要的最小视口与交互复验清单，不伪造截图、证据日志或通过结论。
- 对失败或临界项记录“可见现象 → 原因 → 结构性处理 → 复验视口”；先修最弱关键维度，再在同一证据条件复查。

## 四类定因

- 实现偏差：已有基线但未复用、未对齐、漏状态。
- 基线缺失：缺设计说明、标尺或状态矩阵，实施方只能凭感觉补。
- 审美偏航：功能和状态存在，但层级、比例、节奏、字阶、留白、强调控制失衡。
- 系统破口：局部改动破坏 design system/token/组件契约，跨页不一致。

## 六部看点

- 礼部：层级、排版、留白、对齐、密度、色彩、字重、廉价感；拦大圆角/毛玻璃/渐变/厚阴影/插画/3D 硬撑高级感；工具页误用营销 hero。
- 刑部：关键路径、误触、迷路、卡死；空错慢、禁用、成功、撤销、返回、焦点、键盘、表单校验、危险确认。
- 工部：组件、页面模式、design token、设计规则复用；是否新增平行视觉语言、重复组件、临时样式分支。
- 户部：动效、图层、阴影、模糊、图片、数据密度、感知速度、信息负担、窄屏/低性能可用性。

## 红线

主操作不突出；危险/普通操作难分；去装饰后层级坍塌；宽屏固定窄壳漂在无意义空场；用伪指标、伪客户 / 评价、伪奖项 / 认证或伪截图撑可信度；缺加载/错误/空态/禁用/成功/回退；为高级感牺牲对比度、字号、命中面积或效率；局部好看但不复用系统；动效影响响应或确定感；文案/数字/按钮/表头/导航溢出、裁切、重叠或推动布局跳动。

## 输出要求

不用“感觉不高级”空话；须落到层级、比例、字阶、留白、对比、状态、系统或证据缺口。先报告命中的关键失败；收口指出当前最弱维度、可定位到页面区域 / 状态 / 视口的可见证据、影响、最小修复与复验口径。无设计基线时不让实现方独自背锅。展示型页面勿用后台标准硬压，反之亦然。UI PR 若缺截图、状态样例、响应式或可访问性说明，先列证据缺口；不只凭静态 diff 断“已成品”。

## 快速判词

丑但可修：收层级、字阶、留白、强调数、复用面。能用但别扭：补状态、反馈、回退、焦点流。局部精致整体散：补 design system/token/组件契约。花哨但不高级：减特效，重立视觉骨架。
