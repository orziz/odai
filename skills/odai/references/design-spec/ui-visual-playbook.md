# UI Visual Playbook

## 何时读取

命中网站首页、品牌页、工作台、控制台、仪表盘、设置页、表单流、编辑器壳层、现有前端改版 / 重设计 / 视觉提质 / 去模板味，或用户要求更高级、更像成品时读取。高审美提质同读 `references/design-spec/aesthetic-benchmark.md`。

## 先断三件事

1. 任务性质：新建先立方向；重设计先审旧病灶、复用面与遗留残片。
2. 页面体裁：按首要任务判执行 / 指令、巡检 / 观察、对象检视、叙事、阅读或转化；混合页首屏只留一个主导体裁。品牌 / 叙事页可放大情绪与记忆点，工具页优先扫读、状态、反馈和连续操作。
3. 设计宣言：用户与问题、气质极点、技术 / 性能 / 可访问性 / 成本边界、唯一记忆点。

## 主角与阅读链

- 先锁定用户真正要操作、观察、理解或选择的唯一主角；登录页主角是认证任务，控制台主角是当前状态与可执行动作，对象页主角是真实对象，不能让品牌氛围、标题或装饰抢位。高密列表 / 巡检台的主角可由“当前对象集合 + 最高优先对象及其动作链”共同构成；不能把优先项膨胀成营销式单 hero，反而压低列表主体。
- 用“首要信息 → 解释上下文 → 证据 / 细节 → 主操作 / 延续路径”检查阅读链；缩成缩略图或去掉颜色、插画和特效后，首要信息与主操作仍应可辨。
- 每个主要区域都要承担功能、证据、叙事或感知作用。空白须服务聚焦、分组或节奏；宽屏不得让窄版固定容器漂在无意义空场中，应按内容性质受控扩展、重排或增栏，不能只把整页等比放大。

## 三拨盘

- 构图张力 `1-10`：`1-3` 稳定；`4-7` 偏移 / 错位 / 大小对比；`8-10` 强非对称，仅低频展示面。
- 动效分量 `1-10`：`1-3` 反馈；`4-7` 顺序揭示 / 空间切换；`8-10` 叙事转场，高频工具页慎用。
- 信息密度 `1-10`：`1-3` 展陈；`4-7` 日常产品；`8-10` 高密工具，靠分组、分隔线、字阶，不靠大卡片装箱。

## 成品感五层

任务清楚；信息有主次；栅格、字阶、间距、色彩、圆角、边框、阴影、动效归入少量规则；hover / active / focus / selected / disabled / loading / empty / error / success / 回退齐；只留一个业务相关视觉特征。

## 新建与重设计

新建前端先定首屏任务、色彩承诺、字体系、容器逻辑、记忆点：首屏一主焦点一主 CTA；工具页不用营销 hero、指标墙和无效留白；品牌页可用摄影 / 位图 / 材质，但文字、CTA 和下一段要清楚；主色只承担一个核心角色，语义色另定；暗色模式重定背景层级与焦点可见性；正文行长常控 `65-75ch`。

现有项目重设计按四步：扫代码 / 页面 / 组件库 / token / 邻页语法 → 断排版、颜色、布局、状态、组件、文案、可访问性、性能与遗留 → 清旧容器、旧变体、废 token、旧状态、旧文案、临时 class / props / story / 快照 → 沿既有技术栈薄改。替旧轨时同轮删净，删不稳则记迁移风险。

修复顺序：字体字阶 → 色盘与中性色 → hover / active / focus → 布局间距 → 去模板组件 → 删旧轨 → loading / empty / error / success → 细部抛光。

## 排版与布局

先用字重、字阶、行距、留白定主次，再谈材质；间距要有节奏，不要同 padding、同卡宽、同行高；盒子只在真有分组、承载、状态边界或交互区域时出现；工具页优先工具栏、分隔线、列表、表格、侧栏、抽屉、紧凑状态区；固定格式控件要有稳定尺寸或响应约束，最长文案、按钮、标签、表头、导航项、空 / 错 / 加载态、窄屏、高密数据先验。

## 内容真实性与表达预算

- 优先使用用户提供或仓库已有的真实产品对象、流程、数据、资产和领域术语。未知指标、客户 / 评价、奖项 / 认证、联系方式、法律标识、产品截图或运行数据，不得伪装成可信事实。
- 可提出新字段、筛选项、状态值、枚举、排序规则或运营元数据候选；brief / 仓库未提供的须显式标为提案、待产品确认或占位，不能与已证实内容混写，也不能直接成为验收事实；无必要时省略。
- 字段名已确认不等于该字段的枚举值、颜色映射或业务语义已确认。上游未给值域时使用“最高 / 当前严重度”等抽象称呼；若举 `Critical / High` 一类示例，整组示例与映射必须显式标为待确认提案，不能直接写入视觉规则。
- 上游给出封闭可信内容清单时，交付分开写“已确认内容”和“设计候选（待产品确认）”；所有清单外项目只进候选区，不进入已确认示例或验收。
- 加载、错误、权限不足、已解决、未分派等常见状态也不因“设计上通常需要”而免证；上游未确认时逐项放入候选区或省略，不得写成“必须覆盖”、既定业务规则或验收条件。刷新、筛选、排序等新增能力同理。
- 只留一个与业务有关的主要记忆点。通用渐变、玻璃、光晕、网格、粒子、巨型标题、终端拟态等默认合计不超过一个；额外表达必须各自承担不同且可见的内容、证据、导航或交互职责，并有品牌 / 场景依据。

## 默认禁项

以下风格在没有品牌、场景或用户明确要求支撑时默认避免；若已有明确依据，保留其意图并检查层级、可读性、性能和一致性，不把风格名称本身判成缺陷。

紫蓝 AI 渐变、外发光、玻璃态、彩色竖条、渐变文字、默认居中暗黑 hero、伪科技霓虹边；卡套卡、区块外再包大圆角容器、三等宽卡片循环、FAQ 必手风琴、价格表必三塔；小药丸 / 状态签 / 系统编号 / 伪运行标签堆满首屏；多主色、多主 CTA、多抢眼模块并列；只画理想态，不补加载、空、错、禁用、成功、回退、窄屏。

## 图像先行

宿主支持图像生成，且当前仍缺的是视觉方向时，可按 section 生成参考；图只定方向，仍要回写可实现草案。现有项目重设计不得用图像绕开代码、组件、状态审计。

## 最小交付 / 快检

补齐：页面体裁、唯一主角与阅读链、任务性质、设计宣言、三拨盘、首屏任务与主操作、色彩与字体系、容器 / 间距 / 布局 / 记忆点、内容真实性、状态触感、反模板禁项、重设计病灶与清理顺序、窄屏 / 宽屏响应、是否图像先行。

快检：去掉颜色与特效主次仍成立；小笔电 / 窄栏仍清楚；无三卡片模板、卡套卡、伪技术标签堆叠；只有一个主焦点；最长文案、空错加载、窄屏、高密数据下不挤、不乱、不跳。
