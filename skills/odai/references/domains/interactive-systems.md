# 实时交互系统

游戏、仿真、训练器等含循环、状态、资源与实时反馈，且通用规划 / 设计不足以决策时读取。只用命中部分，不补整套“游戏文档”。

## 循环、内容与数值

- 先说明使用者反复做什么、为何继续、如何成败、怎样回到局外目标；新增功能须说明增强哪个循环、消耗 / 产出什么、是否绕开主体验。
- 数值先定服务的决策，再定公式、区间、曲线、极值和调参点。没有真实基线时把数字标为初始假设，并给样本、指标、阈值和超界后的回调方向。
- 资源设计同时看来源、去向、囤积、断档、追赶和不同强度用户的产消比；局部需求不借机扩展商城、付费、活动或其他系统。
- 关卡、任务、活动与赛季写使用者实际经历的目标、压力、反馈、奖励和再进入理由，同时检查每日负担、追赶、复用与长期产能。

## 界面、反馈与表现

- 高压场景先保危险、目标、资源、冷却、可用 / 禁用和主操作；低压场景再增加比较、构筑、叙事和包装。去掉特效与品质色后，关键判断仍应成立。
- 上游或现有系统给出的正常、锁定、冷却、资源不足、失败、断线 / 恢复、返回与不可逆确认等状态逐项覆盖；未给出的只作待确认候选，不捏造成现状。
- 角色、场景、道具与品牌资产先守识别、观看距离、信息层级和复用；包装不能让角色、标题、奖励和按钮同级抢焦点，也不能掩盖价格、概率或时间。

## 交付与验收

只交当前需要的最小形态：系统关系、体验流程、数值骨架、内容节奏、HUD / 反馈说明或资产规则。验收落到真实输入、压力状态、设备 / 性能边界和可观察结果；不以风格词、功能清单或单点参数代替。

交付前逐项核对当前输入方式。触屏交付必须显式包含两块：

- **触控契约**：UI 明确拇指 / 手掌遮挡安全区、触控热区与按钮间距；客户端明确按下、滑出、抬起、取消 / 中断。若沿用既有操作，也要声明保持原契约，不能用“无遮挡 / 易点”代替。
- **误触复验**：用真实握持压力态覆盖相邻按钮、滑动经过、多指并发和边缘触控，观察可命中与误触场景，并写通过 / 停止条件；没有基线时不编造误触率。

每项必须覆盖或说明不适用。键鼠看密度与快捷路径，手柄看焦点和观看距离；弱网、低端机和高同屏压力下，判定优先于装饰。
