# 通用 UI 与视觉设计

网站、工作台、表单、编辑器或现有前端确需视觉 / 交互专项判断时读取。

## 先定骨架

1. 判断新建或重设计，以及首要任务：执行、巡检、对象检视、阅读、叙事、转化或其他真实目的；混合页明确主次，不用模板体裁替用户决定。
2. 锁定主角与阅读链：首要信息 → 证据 → 主操作 → 延续。工具页不用营销 hero、KPI 墙或空场抢主任务。
3. 用少量规则组织栅格、字阶、间距、颜色、圆角、边框、阴影和动效；盒子只表达真实分组、状态或交互边界。
4. 从品牌、内容、场景、平台和既有系统选择视觉语言；每个显著视觉手段都应服务层级、识别或反馈，不因流行样式自动加入或禁止。

## 真实性与压力态

- 优先用真实对象、数据、流程和资产。无来源客户、指标、SLA、评价、奖项、联系方式、截图和运行数据须删除、占位或待确认；候选不冒充既有规则。
- 相关 hover、focus、active、selected、disabled、loading、empty、error、success 和回退须可辨；不补触达不到的状态。
- 最长文案、窄 / 宽屏、高密度、键盘和低性能按风险取最小判别集。宽屏不留窄壳空场，触屏不牺牲命中和误触安全。
- 去掉颜色、插画和特效后，主次、危险、收益和主操作仍成立；可访问性和性能不为“高级感”让路。

## 新建、重设计与验收

新建先定首屏任务、内容骨架、视觉语言、组件规则和关键反馈。重设计先扫页面、组件库、token、邻页和旧轨，再按当前目标修；旧轨能安全替换才删除，不能就明确兼容与迁移风险。

复核主任务 / 主操作、目标视口、内容可信度、组件一致性、关键状态 / 响应式、焦点 / 对比 / 命中；任一关键维度阻断都不能靠精致细节平均。

浏览器可用时以真实页面、viewport 和核心交互为证据；不可用时只报静态风险，并给视口 / 状态 / 操作及通过条件，不伪造视觉验收。
