# 规划与决策

目标宽泛、约束冲突、存在多条实质不同路线，或实施前仍缺关键决定时读取。目标是形成足以行动的决定，不是产出最完整的规格。

## 形成决定

1. 用一句话写清谁在什么场景得到什么可观察结果；区分结果、手段和非目标。
2. 从用户原话、项目事实、现有行为和权威材料恢复硬约束、依赖、已确认决定与不可接受结果。
3. 只保留会改变当前路线的开放项。可自行查证的先查证；必须由用户取舍的，给真实不同的选项、推荐与改选条件。
4. 比较候选对结果的解决程度、改动面、风险、维护、可逆性和依赖；没有实质差异就不制造多方案。
5. 裁成当前交付所需的最小规格：目标、范围、关键行为、非目标、验收、风险和下一动作。

决定已足以行动时停止规划。用户要落地且授权、证据与边界已足够时继续交接或实施；不以规格文档存在冒充成事。

## 开放项与数值

- 疑似误写、证据不足或要求冲突时保留原话，分开“当前理解 / 已确认 / 提案 / 待定”，不把推荐默认值写成用户决定。
- 数值先写它服务的决策、依据、敏感性、极值和验证方式。无生产基线时不伪装成生产值；用户要方案时，仍交付由现有目标反推、可计算、可回退且明确标为待验证的最佳候选，把后续校准证据与上线门分开。
- 功能、内容、经济、流程和运营都先描述输入、状态变化、输出、失败 / 回退与观察信号；不因领域名另开工作流。
