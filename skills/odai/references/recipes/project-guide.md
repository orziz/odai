# Project Guide

用户要求归纳整个项目的目标、结构、命令、约束、命名、术语或协作方式时读取。默认更新项目级说明文档，不修改无关业务代码。

1. 先读 README、规则文件、关键配置、目录和现有文档，再选择主文档。
2. 区分用户确认、代码可见、已验证和仍待确认的内容；拿不稳不写成项目规则。
3. 优先更新现有 README、PROJECT 或 docs 入口，不复制平行说明。
4. 文档与代码冲突时标明各自来源和影响，不任意选择一方。
5. 高频歧义词可进入词汇表；只有高代价、跨领域或跨系统且需要长期追溯的架构决策才建议 ADR。

按受众裁剪目标、技术栈、目录、关键命令、高风险区域、术语、协作方式和验证入口；先给可用结果，再给证据和真实未决项。
