# Ribao

用户要求日报、工作总结、commit message 或 PR message 时读取。

## 取证

输入优先级：用户明确描述 → 指定范围的 git 记录和当前改动 → 相关文件 → 对话上下文。用户未给内容时，只在当前项目内查看必要的 log、status、diff stat 和相关 diff；避免把大 diff 全部灌入上下文。

## 输出

- 核心回答做了什么、为什么做、结果或价值是什么。
- 默认简洁，每条尽量包含“动作 + 目的 / 结果”，不写流水账，不机械罗列文件名。
- 日报突出当天可核实产出；commit message 凝练描述本次提交；PR message 可补背景、主要改动和验证。
- 不加无用引导语，不编造未被用户或项目证据支持的事实。
- 没有可核实内容时直说，不用通用套话填充。

语言和语气遵循用户要求与项目惯例；真实性不因轻松表达而降低。
