# System Numeric Commercial Playbook

## 何时读取

命中以下任一主题时，读本文件：

- 系统结构、循环、成长、功能耦合
- 数值框架、平衡、成长曲线、难度曲线
- 资源 / 经济循环、货币分层、产消关系
- 商业化、付费结构、价值包装、公平性边界

## 最小顺序

1. 先定核心体验、目标玩家和胜负 / 成长目标。
2. 再拆系统循环、资源流向、数值区间和商业边界。
3. 再补关键风险：破坏平衡点、资源断档、成长塌陷、付费压迫感、公平性争议。
4. 末定交付：系统方案、数值框架、经济说明或商业化方案。

## 系统策划短框架

至少补齐：玩家局内外反复做什么；胜负/完成条件；系统如何互相喂数或限制；成长结构如何制造长期目标；何处最易跳过主循环形成破局解。

默认不只写功能列表；要写出功能与循环之关系。

## 数值与经济短框架

至少补齐：关键资源、来源/去向；角色/装备/关卡/对局关键数值轴；成长与难度曲线随时长/版本如何变；正常/重度/付费玩家产消比是否失衡；哪些极值会打穿平衡或内容消耗速度。

默认不只给单点参数；要给为何如此定、后续如何调。

## 商业化短框架

至少补齐：商业目标（首购、ARPPU、留存带动、赛季转化、版本回收）；付费点与货币体系如何挂循环；价值包装如何配成长/活动节奏；不可越界（公平性、社交压力、强制付费感、付费前置卡点）；活动/赛季/版本更新时商业结构如何迭代且不压垮原系统。

默认不把商城界面、商品文案或单次促销当商业模型本体。

## 最小交付

按主矛盾选：

- 系统方案：目标、循环、系统关系、成长结构、风险
- 数值框架：关键公式、区间、曲线、调参点、验证口径
- 经济说明：资源层级、产消链路、风险点、版本影响
- 商业化方案：付费结构、价值包装、边界、节奏与风险
