# Content Level LiveOps Playbook

## 何时读取

命中以下任一主题时，读本文件：

- 关卡、副本、任务、章节、引导与节奏安排
- 活动、版本、赛季、长期运营和内容投放
- 叙事、世界观、角色内容、文本结构和题材承接
- 内容产能、复用机制和版本更新节奏

## 最小顺序

1. 先定目标：本次内容要拉什么指标、制造什么体验、解决什么版本问题。
2. 再定结构：章节 / 关卡 / 活动 / 赛季如何分层，奖励和门槛怎么挂。
3. 再补节奏：首次体验、中期重复、后期追赶、版本更新和资源回收。
4. 末补产能：哪些可复用，哪些必须新做，长期成本是否可承。

## 关卡与任务短框架

至少补齐：玩家目标、失败条件、反馈节奏；关卡结构、事件编排、难度曲线、奖励投放；新手引导、教学节点、卡点保护；可复用模板与一次性内容比例。

默认不只写地图或主题概念；要写玩家实际经历之节奏。

## 活动、版本与赛季短框架

至少补齐：本次活动/版本/赛季拉何指标或修何问题；内容开放节奏、资源回收、奖励结构、疲劳控制；与原系统/商业/数值节奏的耦合；对老/新/回流玩家影响；版本结束后何者可复用、何者成技术或内容债。

默认不把活动主题包装当活动方案本体。

## 叙事、世界观与内容短框架

至少补齐：题材定位与内容功能（沉浸、转化、角色认同、版本承接）；世界规则、角色关系、剧情推进、分支控制；文本/演出/立绘/任务/事件系统如何协同；内容产能、复用机制、跨版本延展。

默认不只交设定词条；要说明这些设定如何服务玩法、版本与商业目标。

## 最小交付

按主矛盾选：

- 关卡 / 副本方案：目标、结构、事件、节奏、奖励、风险
- 活动 / 版本 / 赛季方案：指标目标、内容节奏、资源回收、复用与风险
- 叙事 / 世界观草案：题材定位、角色关系、剧情功能、产能与承接
- 新手引导方案：教学节点、卡点保护、反馈节奏和早期留存目标
