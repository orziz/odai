# Agent、模型与执行下放治理

## 何时读取

触发条件只按 SKILL.md「低频规则触发表」判定，并在决定采用及实际调用前读取，同时外显「触发：`agent-governance`｜条件：『<命中关键词>』」。未外显不得声称已调用或下放。这里只治理真实 agent / 模型调用与下放；待冻结结论的独立挑战和合议征询归 `references/dao/decision-challenge.md`，提问、实施准入和真实性仍归 `references/dao/interaction-contract.md`。

Agent 可用来扩大证据面、生成候选路径 / 候选实现、并行只读审查、执行已冻结任务或做独立挑战；不得把 agent 数量当可靠性本身。候选生成可积极，采纳必须保守；未被主流程用证据复核的内容只作线索。

## 采用原则

- 只有宿主真实暴露相应能力且上层允许时，才能声称并行或调用了 agent / model；否则在主流程内反证。不得用同模型副本或只换提示词冒充多模型。
- 能由主流程低成本读证、跑验证或查文档解决的，不下放。
- 仅当任务可切分、边界清晰且收益明显高于冷启动与交接成本时使用 agent。同模型仅为隔离上下文的调用走本门，不得包装成多模型或低成本执行下放。
- 优先让 agent 产出可比较、可复验、可丢弃的候选；主流程按验收口径采纳可证部分，丢弃未证或越界部分，不为保留产出而改边界。

## 执行下放

方案 / 执行单已锁死且无决策级判断时，可把重执行下放。四条全中才下放：

1. 宿主支持真实 agent / 模型档；若涉及成本档，高低必须有宿主标签、现成价目或用户授权作证。成本档不明且会影响下放选择时，fail fast：不得长搜价目或凭记忆比价；结构化列出宿主可用档、未知项与退化路径请用户选定；未选定前不得按成本下放。
2. 范围、行为、验收、非目标已锁定，实施准入与确认已由主流程完成。
3. 执行量足以覆盖冷启动、交接与复核开销。
4. 主流程能读产物并运行必要验证。

下发包必须含冻结题面、输入材料、允许动作、禁止越线项、验收证据、停手回交纪律。涉及代码 / 配置 / 脚本写入时，实际写入方必须持有同版 `references/dao/interaction-contract.md` 与 `references/modules/implement-code.md` 全文，不得用摘要替代；无法确保就不得下放该写入段。

## 子 agent 通用边界

适用于执行下放、并行分析、审查和挑战席；各专项文件只能叠加，不得削弱：

- 子 agent 无用户通道，产出只是回交主流程的数据，不得冻结方案或宣布完成。
- 出现待确认点、冲突点、留中待问，或新证据触及范围、行为、验收时，立即停手回交；不得自行问用户或猜测后继续。
- 不再派生子 agent，不自行扩增轮次。
- 范围、设计、验收裁决只属主流程。
- 主流程复核（读 / 跑）通过后才可采纳，不得把子 agent 的自报完成当证据。
- 子 agent 输出必须区分已读材料、已跑动作、观察事实、推断、建议和未验证缺口；无法复验的结论只能当线索。
