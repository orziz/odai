# Agent、模型与执行下放治理

## 何时读取

触发条件只按 SKILL.md「低频规则触发表」判定，并在决定采用及实际调用前读取，同时外显一行「触发：`agent-governance`｜条件：『<命中关键词>』」，未外显不得声称已调用或下放；只在主流程内做普通分析或反证且未命中总表时不读。这里只治理真实 agent / 模型调用与下放；待冻结结论的独立挑战和合议征询归 `references/dao/decision-challenge.md`，提问、实施准入和真实性仍归 `references/dao/interaction-contract.md`。

## 工具与进度克制

- 只有宿主真实暴露相应能力且上层允许时，才能声称并行或调用了 agent / model；否则在主流程内反证。不得用同模型副本或只换提示词冒充多模型。
- 不得仅为节省主会话上下文拉子 agent；仅当任务可切分、边界清晰且收益明显高于冷启动与交接成本时使用。同模型仅为隔离上下文的调用走本门，不得包装成费率套利。

## 执行下放（费率套利）

方案 / 执行单已锁死且无决策级判断时，可把重执行下放给更低成本执行档。四条全中才下放：

1. 宿主支持按模型档拉 agent，且成本高低有宿主标签、价目或用户授权作证；不得凭训练记忆猜。其余条件全中但成本档不明时，结构化列出可用档请用户选定；用户不选、无更低成本档或宿主不支持时留主流程。
2. 范围、行为、验收、非目标已锁定，实施准入与确认已由主流程完成。
3. 执行量足以覆盖冷启动、交接与复核开销。
4. 主流程能通过读产物及必要运行验证结果。

下发包必须含实施准入、停手回交纪律与冻结题面。涉及代码 / 配置 / 脚本写入时，实际写入方必须持有同版 `references/dao/interaction-contract.md` 与 `references/modules/implement-code.md` 全文，不得用摘要替代；无法确保就不得下放该写入段。模型档与 agent 仅在真实拉起时回报，不得用同档副本冒充套利。

## 子 agent 通用边界

适用于执行下放、并行分析、审查和挑战席；各专项文件只能叠加，不得削弱：

- 子 agent 无用户通道，产出只是回交主流程的数据，不得冻结方案或宣布完成。
- 出现待确认点、冲突点、留中待问，或新证据触及范围、行为、验收时，立即停手回交；不得自行问用户或猜测后继续。
- 不再派生子 agent，不自行扩增轮次。
- 范围、设计、验收裁决只属主流程。
- 主流程复核（读 / 跑）通过后才可采纳，不得把子 agent 的自报完成当证据。
