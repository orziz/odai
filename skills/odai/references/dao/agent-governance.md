# 模型与 Agent 治理

本文件是真实模型路由、agent 调用和执行下放的 owner。用户显式要求模型 / agent / 成本路由，或 `references/dao/agent-routing-gate.md` 命中时读取。

## 能力真实性

- **主流程路由**：宿主真实暴露的当前模型切换或等价 model override。subagent / 新线程指定模型不等于主流程切模型。
- **Agent 调用**：宿主真实暴露的可启动、交接和回收结果的执行单元。不用同模型副本或只换提示词冒充异构多模型。
- 未真实暴露能力时，留在主流程，不伪称切模型、并行或调用 agent。

## 选择执行形态

1. **主流程直办**：主流程低成本可解、任务不可切分、需求未稳或交接收益不足。
2. **主流程切模型**：宿主已暴露路由能力，且当前任务与模型标签、边界、成本和验证能力匹配。
3. **Agent 候选 / 执行**：用户明确要求，或已命中 `references/dao/agent-routing-gate.md` 的可观察收益条件；范围与验收必须已稳，且主流程能够复核。
4. **独立挑战 / 合议**：待冻结结论高代价或难回退时，先读 `references/dao/decision-challenge.md`；用户显式激活合议时再读 `references/dao/consensus-mode.md`。

只为节省成本时，优先主流程切模型；宿主只有 subagent model override 时，如实标成 agent 退化路径。

## 能力匹配与上调

模型路由不只用于降本：

- 同一路径重复失败时，先补最小证据或换正交诊断 / 实现方向止损；换向后仍在同一判别性验收失败，或已有上下文、模态、工具 / 推理能力缺口的直接证据时，才判为能力不匹配并停止无新增信息的重试。
- 宿主明确暴露更高能力标签时，优先建议或切换主流程到最小充分的更高档。
- 只有 subagent / 新线程支持更高能力 override，且验收结果可由当前主流程用独立工具或判别性证据复核时，才把冻结子任务交给该 agent，并如实标成退化路径。
- 主流程不能可靠复核时，仍应建议切换主模型，不借 agent 自报绕过能力缺口。

下一步路由只要提到 subagent / 新线程 override，就必须在同一输出中同时写明“这是退化路径”与“仅在当前主流程能独立复核时允许”；不能完整带出这两个限定时，不提供该退化选项，只保留上游主模型切换。

能力上调不扩大权限、范围、成本上限或验收权威。只依据宿主标签、真实失败证据和用户授权判断，不凭模型名、版本号或训练记忆宣称谁更强；没有可证更高档时，列明能力缺口与最小解除条件，不伪称已切换。

选择 agent 后，命中以下任一项必须读 `references/dao/execution-orchestration.md`：

- 两个以上任务、任务依赖或跨压缩恢复。
- 预计产生大 diff、长扫描报告或多轮日志。
- 只读审查确认多个问题，准备进入修复波。

一次性小任务和短回交不读。

## 成本与候选

- 只使用宿主标签、当前价目或用户授权证明能力 / 成本档，不凭模型名或训练记忆比价。
- 用户显式要求按成本、指定模型或 agent，但标签 / 候选不足以唯一选择时，fail fast：列「宿主可用能力」「未知项」「退化路径」，给一个不依赖虚构成本档的推荐后让用户选择或授权按推荐执行。无价目或成本标签时，不得点名某模型并称其便宜 / 低成本。
- 非用户显式要求时，候选选择不影响授权、成本上限或停止条件，可按最小充分能力自主选择，不为路由阻断任务。

## 下放契约

最小下发包只包含 agent 完成任务真正需要的内容：

- 目标与输入材料。
- 允许动作、范围、保持不变项与不可越线项。
- 验收证据、停止条件和回交形式。
- 与本任务直接相关的 owner 规则或路径；不把整段聊天或无关 skill 文件全量下发。

涉及代码、配置或脚本写入时，brief 必须外显四行：

```text
源契约：<同版 references/dao/interaction-contract.md 精确路径>
实现规则：<同版 references/modules/implement-code.md 精确路径>
读前门：写入 agent 首次写入前完整读取两者；摘要不得替代源规则
退化：无法访问或确认同版源时，不得写入或下放该写入段
```

## Agent 权限

Agent 在冻结范围内可自主：

- 选择局部实现细节、证据顺序、工具和验证组合。
- 做可逆、不改变用户可见契约的局部决策。
- 在宿主允许、主流程明确授权且已给深度 / 成本 / 停止上限时，再下放可独立子任务；否则不递归派生。

Agent 必须停手回交：

- 新证据动摇用户目标、范围、授权、验收、成本上限或不可接受结果。
- 需要不可逆、生产、外部系统或额外费用动作。
- 无法达到验收或命中停止条件。

主流程必须用可复验证据采纳 agent 产出，不把 agent 数量或自报完成当可靠性。实际调用后，只按真实发生的对象、边界、结果和验证回报。
