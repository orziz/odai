# 模型、Agent 与执行协作

用户点名模型、agent、并行、成本、多模型复核，或独立执行 / 挑战有净收益时读取。

## 是否下放

优先主流程直办。仅当宿主提供真实能力、边界与验收稳定、主流程能够复核，且收益高于冷启动、交接和冲突成本时使用 agent。

适合下放：无写冲突的独立分支、明确的大扫描、高代价结论反证、冻结输入的重复执行。上下文强耦合、短任务、写冲突或不可复核则直办。

不要虚构模型、价格、能力、并行或调用。成本只据宿主信息；能力未知就说明，不凭名称或印象定强弱。

## 交接与采纳

最小 brief 含目标 / 输入、范围 / 禁区、保持项、允许动作、验收、停止和回交形式；只发所需材料，不复制聊天或整套 skill。

代码、配置或脚本写入下放时，brief 还必须外显同版读前门：

```text
源契约：<同版 skills/odai/SKILL.md 的可访问路径>
实现规则：<同版 references/capabilities/implement-code.md 的可访问路径>
读前门：写入 agent 首次写入前完整读取两者；摘要不得替代源规则
退化：无法访问或确认同版源时，不得下放写入，保持只读回交
```

Agent 可在冻结范围内定局部做法；新证据改变目标、授权、范围、成本或风险就停手回交。主流程按原始产物、diff、工具结果或独立证据采纳，不按 agent 数量或自报完成。

并行任务隔离写域与依赖，上游先验证。长任务按 `references/dao/continuity.md`；审查只读，修复另行授权。

修复波先去重并冻结 finding ID；Review agent 始终 `READ_ONLY`，Fix agent 只接冻结清单，主流程逐项复验后才关闭。新发现进下一波。

## 挑战与换挡

- 高代价、难回退或多路近似的决定，冻结前做一次独立挑战或显式反证；按证据裁决，不按多数票。
- 用户明确要求合议、多模型复核或重型压力测试时，再读 `references/techniques/consensus.md`。
- 同向尝试无新信息时，换正交证据或回查问题；只有明确能力缺口且宿主有更合适能力时才换模型。
