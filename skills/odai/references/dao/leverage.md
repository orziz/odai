# 借力与协作

需要外部能力、项目规则、附件、领域资料、agent 或多模型协作时读取。借力是成事手段，不是进度证据；当前能力已能闭合结果时，不为发现、增强或完整感额外加载。

## 先组合能力

1. 先服从宿主、项目规则与用户指定资源；使用宿主已提供且直接匹配的文档、表格、演示、PDF、浏览器或图像等能力。
2. 内部能力管通用工艺，`references/domains/` 只补当前所缺专业知识。外部工艺须直接相关、补足有效知识或工具，并兼容当前版本与约束；空泛、重复、陈旧或不可验证者不加载。
3. 从对象、术语、文件、平台和现有方式自动推断领域，不让用户选择内部包；仅当差异会改变目标、验收或高代价路线且证据不足时询问。
4. 非宿主可调用的文本只作资料，不冒充已调用技能；仅在项目根高信号规则、用户点名路径或宿主暴露位置查找，不扫描 home、猜路径或读取秘密。

项目根 `.odai/local.md`、宿主暴露的 `odai-local` 或用户点名偏好可作叠加；项目层高于用户层，两者服从 canonical odai、宿主和当前用户指令。读取不等于写入，一次观察不固化。

## 再决定是否下放

优先主流程直办。仅当宿主提供真实能力、边界与验收稳定、主流程能够复核，且收益高于冷启动、交接和冲突成本时使用 agent。

适合下放：无写冲突的独立分支、明确的大扫描、高代价结论反证、冻结输入的重复执行。上下文强耦合、短任务、写冲突或不可复核则直办。不要虚构模型、价格、能力、并行或调用；成本只据宿主信息，能力未知就说明。

最小 brief 只含当前所需的事、事实 / 输入、范围 / 禁区、保持项、允许动作、验成、停止和回交形式。不要复制整段聊天或无关 skill。

写入下放前，确认执行者已获得同版 `SKILL.md` 与 `references/capabilities/delivery.md` 的有效规则。宿主已完整携带就不重复加载；未携带则在首次写入前读取可访问源，摘要不能替代源规则。无法访问或确认同版时降为只读回交，不虚构受约束的写入协作。

Agent 可在冻结范围内定局部做法；新证据改变目标、授权、范围、成本或风险就停手回交。主流程按原始产物、diff、工具结果或独立证据采纳，不按席位数量或自报完成。

并行任务隔离写域与依赖，上游先验证。审查 agent 始终只读；修复波先去重并冻结 finding ID，fix agent 只接冻结清单，主流程逐项复验后才关闭。新发现进下一波。

高代价、难回退或多路近似的决定，冻结前做一次独立挑战或显式反证；按证据裁决，不按多数票。用户明确要求合议、多模型复核或重型压力测试时，再读 `references/techniques/consensus.md`。

## 演进而不自改

任务中发现会跨任务复发的规则空白、冲突或陈旧工艺时，先完成当前“事”，再提出有证据的改进候选；不自动改 skill，不混入业务待办。用户采纳后，项目规则写 `.odai/local.md`；提升进 canonical source 须另获维护授权，提升后才删重复叠加。候选数量由真实缺口决定，不为“持续改进”凑提案。
