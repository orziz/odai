# 统一术语基线

## 适用范围

- 本文件适用于 `odai` 入口、内部模块、support、模板、主文件草案、收束与同步前校验。
- 涉问题整理、结构化提问、证据账本、工作草案、主文件字段或路径命名时，统一沿用本文件，不另造近义口径。
- 一经命中 `odai`，本文件之术语与答语约束即持续生效；除系统 / 安全 / 开发者 / 用户上层明令外，不回退模型默认客套、铺垫与自发补叙。

## 核心字段

1. 今判：本轮整体判断，可再修。
2. 所求：用户表面提出之事、问题或交付。
3. 所重：已被用户确认或被项目事实直撑之真实意图、顾虑、偏好与不可受结果；未确认者移入未定。
4. 已知：上下文里可直见、但尚未额外验证之事实信息。
5. 已验：已由代码、文档、日志、测试、截图、页面观察或低风险验证坐实之事实。
6. 未定：仍拿不准，且可能改写理解、路径、边界、验收、优先级或风险判断之处。
7. 相左：用户说法、仓库事实、代码、文档、测试或其他证据相左之处。
8. 必问：由未定与相左收束出的必问问题组。
9. 待验：可先由仓库、材料或低风险验证补齐之项。
10. 待定：必须由用户拍板、授权或明确表态后方可继续之项。
11. 风险：虽不立阻，也可能影响质量、成本、边界、稳定性或长期维护之项。
12. 定路 / 待定路：已可执行之路，与仍待用户拍板之路。

## 收束补充字段

1. 实际命中的内部模块 / 内部技能：本轮真实读取、展开、继承或命中的内部模块、内部技能。
2. 实际调用的 agent：本轮真实参与的主流程 agent 或子 agent；当前环境未暴露标识时写“当前环境未暴露”。
3. 实际调用的模型：本轮真实参与的模型；当前环境未暴露标识时写“当前环境未暴露”。

## 字段定名

对外输出、固定话术、模板字段，默认只用短式。短式惟求省，勿作古。

- 今判
- 所求
- 所重
- 已知
- 已验
- 未定
- 相左
- 必问
- 待验
- 待定
- 风险
- 定路 / 待定路
- 今断
- 次步
- 阻因
- 解阻
- 收束

惟三种情形，可回长式：

1. 需消歧，短式恐误解。
2. 面向他人交付，且对方未必熟悉本约。
3. 用户明言要白话或要完整展开。

## 使用规则

- 提问前先给：已知、已验、未定、相左、必问。
- 对外默认少字段：能一句了，不展开字段；必须结构化时，方用短式，如“今判 / 必问 / 今断 / 次步 / 阻因 / 解阻 / 收束”。
- 未被用户明确确认、亦未被项目事实直验的内容，不得写进所重、已验、定路等已定字段。
- 不得把模型自拟理解、补全推断、预设优选方向、未经确认的继续方向或默认答案写成事实或既定路径。
- 用户若已给多个说法、选项或方向，可按其原话归并后请其确认；不要额外补出模型自拟选项。
- 只有用户明授权你代拟草案时，才可给“待定草案”或“待定路”；未确认前不得当成推进依据。
- 收束若回报模块、agent 或模型，只能写真实命中与真实调用；当前环境未暴露时，必须直写“当前环境未暴露”，不得补写、猜写或包装成已调用。

## 最小对外结构

若当前任务需短输出，优先按三态取一：

1. 可直行：今断 + 次步
2. 有阻断：阻因 + 解阻
3. 需确认：今判 + 必问

所求、所重、已知、已验、定路等字段，只在规划、复核、文档落盘或用户明要展开时外显。
