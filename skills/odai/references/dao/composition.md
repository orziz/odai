# 专项能力与项目规则组合

需要外部能力、项目规则、附件、领域资料或叠加层时读取；odai 管目标、边界与结果。

## 组合顺序

1. 先服从宿主、项目规则及用户指定资源。
2. 使用宿主已提供且直接匹配的技能，如文档、表格、演示、PDF、浏览器或图像能力。
3. 内部能力管通用工艺，`references/domains/` 只补当前所缺专业知识，不为完整加载整包。
4. 非宿主可调用的文本只作资料，不冒充已调用技能；仅在项目根高信号规则、用户点名路径或宿主暴露位置查找，不扫描 home、猜路径或读秘密。

外部工艺须直接相关、补足有效知识 / 工具且兼容当前版本与约束；空泛、重复、陈旧或不可验证者不加载。

从对象、术语、文件、平台和现有方式自动推断领域，不让用户选择内部或领域包；仅当差异会改变目标、验收或高代价路线且证据不足时询问。

项目根 `.odai/local.md`、宿主暴露的 `odai-local` 或用户点名偏好可作叠加；项目层高于用户层，两者服从 canonical odai、宿主和当前用户指令。读取不等于写入，一次观察不固化。

## 演进闭环

非直达任务发现会跨任务复发的规则空白、冲突或陈旧工艺时，任务完成后最多提出一条候选：目标规则 / 文件、改法、证据、风险；不自动改 skill，不混入业务待办。用户明确采纳后，项目规则写 `.odai/local.md`，跨项目偏好只写宿主暴露或用户点名层；目标不明只回报。提升进 canonical source 须另获维护授权，提升后才删重复叠加。
