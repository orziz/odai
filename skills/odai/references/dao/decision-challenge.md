# 高代价决策的独立挑战与合议征询

## 何时读取

触发条件只按 SKILL.md「低频规则触发表」判定。命中时，须在结论最终冻结及据此实施前读取，并外显一行「触发：`decision-challenge`｜条件：『<命中关键词>』」，未外显不得进入挑战或冻结；本文件只治理待冻结结论的独立挑战与合议升档。实际调用 agent / 模型、并行或执行下放时另按总表读取 `references/dao/agent-governance.md`。

## 自动一席挑战

1. 命中读取条件时，最终冻结前必须做 1 席独立挑战；不允许主流程自评后直接冻结，也不再另做成本收益门判断。
2. 挑战席围绕同一已锁定题面与待冻结结论找反例、漏判、备路与不可接受结果，不能复述主流程；主流程须显式收口为采纳、部分采纳或驳回并说明理由。
3. 挑战输入只含已锁定题面、范围、待冻结结论和完整正反证据，不得暗示预期结论、要求忽略某类问题或隐藏反面材料。本条同样约束 `review-sslb` 锦衣卫与合议各席。
4. 宿主有异构模型时优先异构；只有同系 agent 时用同模型独立挑战且切入点必须不同；无 agent 能力时由主流程显式反证。实际调用 agent / 模型前先读 `references/dao/agent-governance.md`，不得伪称调用。

## N 席合议升档

1. 用户显式说 `增强模式` / `合议` / `多模型复核` / `开合议` 或等价表达时，读取 `references/dao/consensus-mode.md` 进入 N 席重型版；否则先完成自动 1 席挑战及主流程收口。
2. 未显式激活时，自动 1 席完成并收口后，若决策仍同时满足高 stakes、真多路、难回退，且 1 席不足以覆盖反例、备路和不可接受结果，主流程必须征询是否升为 N 席合议；用户同意才读取 `references/dao/consensus-mode.md`，拒绝则沿用 1 席收口。征询只暂停 N 席与最终冻结，不抹去已经完成的一席挑战。
3. 同一独立决策点在本次请求内只征询一次。用户拒绝后，题面、风险、备选路线与宿主能力均无新证据时不得复提；后续同一会话出现新请求、新决策点，或新证据使原决策重新满足升档条件时，重新判定并可再次征询。同一请求内多个决策点同时命中时，优先成组一次问清。
