# 结果总结与技能改进候选

## 展示层级

按完整总结 → 标准总结 → 轻量附录依次判定，命中即停，不得用低档覆盖高档：

1. **完整总结**：复杂任务、严格模式、blocked、权限风险、跨模块主路冻结、正式审查或用户追问时，完整回报主控模块、展开模块、support files / 脚本、未完成项、风险与下一步。正式审查指用户明确要求完整 / 正式模式，或结论将作为合并、发布、安全或全仓 / 全目录准入门；普通日常 diff / 文档审查不自动算正式。
2. **标准总结**：涉及文档回写、同步、审查、实现、测试、模块切换或维护时，至少列主控模块、实际展开模块、实际读取的 support files / 脚本与验证结果。
3. **轻量附录**：未命中前两档的普通小任务、单模块只读结论或低风险局部动作，且未启用严格模式或跨模块裁决时，压成一句“采用 / 读取 / 验证”。

主文件已落盘的标准 / 完整总结，须保证主文件自包含；结尾只注明“本任务已沉淀至主文件，任意会话说「继续」均可接续”，不指挥用户管理会话。轻量附录不附此句。

主文件已落盘时，收口未完成可恢复项按 `references/dao/dao-shu-fa-playbook.md` 的「后续队列」写回当前任务主文件；不得把未确认建议、风险观察或技能改进候选自动写成可执行待办，也不得用入队代替当前范围内仍可继续完成的动作。若本轮新增、更新或完成队列项，结果总结必须给出可点击主文件路径、明确提示可编辑 `## 后续队列`，并附队列状态摘要，方便用户稍后查看、勾选、改状态或维护。

后续队列无法落盘时，按 `references/dao/dao-shu-fa-playbook.md` 输出「未落盘后续队列」；该段必须可被上下文压缩完整保留，并提醒后续优先写入主文件或让用户确认丢弃。

统一回报字段：主控模块（通常 `dao`，对外称 `道`）、实际展开模块、实际读取的 support files / 脚本、实际调用的 agent / 模型、实际借力的外部技能。agent、模型、外部技能仅在真实调用时回报；标识未暴露时写“当前环境未暴露”。未读取、未参与判断的模块不得列入。

标准总结默认把上述审计字段合并成一行，不在正文逐项重复；只有完整总结或字段本身需要解释时才展开。合并只改展示，不得漏报或虚报。

## 技能改进候选

标准或完整总结时，若本轮暴露出换个任务仍会复发的规则空白、冲突或 stale 规则，可附一条最关键候选：目标规则 / 文件、改法（新增 / 改写 / 废弃）、证据、confidence、risk、验证方式。候选只提议，不自动改 skill，不得触碰精神内核与既定纪律。

判据是可泛化而非发生次数；一次但明显可复发即可提，纯本任务偶发或普通功能建议不提，轻量附录不触发。用户采纳后按 `references/dao/local-overlay.md` 默认写本地叠加层；是否提升进源是维护者另行决定。

真实性硬法统一按 `references/dao/interaction-contract.md`；本文件不得另写宽松例外。
