# 合议增强模式（用户激活）

## 何时读取

- 这是 `references/dao/decision-challenge.md` 自动 1 席挑战的用户激活重型版：升级为 N 席多模型合议与按模型强弱分工。
- 仅在用户显性激活时读取并执行；默认路径不加载本文件，不产生本模式开销。
- 前提是宿主真暴露多模型（如 Copilot 内 GPT / Claude / Gemini，及用户自挂的 DS / GLM 等）。激活了但宿主给不出，按「能力不足时」处理，不伪造。

## 怎么算激活

- 用户显式说出激活词：`增强模式` / `合议` / `多模型复核` / `开合议`，或等价明确表达。
- 或主流程依 `references/dao/decision-challenge.md` 的征询式升档规则询问、且用户同意——等同显式激活。
- 可顺带指定席位数与模型，如「合议 4 席 用 gemini ds gpt glm」；未指定时用下面的默认配置。
- 未显式激活、且未经上述征询同意，一律不进入本模式；不得因任务“看起来重要”就自动开。其余自动场景由 `references/dao/decision-challenge.md` 的 1 席兜底覆盖。

## 席位与配模

- 默认 3 席，席位数由用户指定时以用户为准。
- 配模：宿主能给就保证 ≥2 种异构模型，最多 1 席与主流程同模；不得用同模型副本、只换 agent 名或只换提示词冒充异构。
- 用户点名模型按点名；点名模型数与席位数对不上时，先问清再开，不自行补位。
- 下发前必须确认点名模型在当前宿主真能派起；派不起的先剔除或回问，不得先宣称已合议、再把失败藏到结尾；真实性与能力不足处理按交互契约执行。

## 模型倾向（经验偏好，须有当前宿主证据）

下表只登记本仓库经验偏好，不构成模型能力、成本或可用性的事实。只有当前宿主真实暴露对应模型，且所需能力或成本判断有当前标签、价目、实测或用户授权支撑时才采用；否则只按任务角色选当前可用模型，不凭本表猜测。用户可逐任务覆盖：

| 任务性质 | 倾向模型（首选家族，无则取可用替代） |
|---|---|
| 规划 / 谋定 / 方向裁决 | Claude |
| 设计 / 创意 / 视觉 | Gemini |
| 复核 / 审查 / 找漏 | GPT、GLM |
| 代码实现 / 落地 | 质量敏感可参考 Claude；大体量低风险任务取宿主已证实的低成本可用档（证实为 DS 时可用 DS） |
| 主流程收束 / 裁决 | 当前宿主主模型 |

覆盖优先级：用户本轮点名 > 有当前证据支撑的本表偏好 > 主流程主模型兜底。成本敏感的大体量子任务先按 `references/dao/agent-governance.md` 核实成本档；只有 DS 被当前宿主证实为更低成本且满足任务要求时才优先。首选家族不可用时，取当前证据支持的可用替代，并优先选与主流程不同家族以保异构；发生替代时在结果总结写明。

## 覆盖范围与流程

本模式叠加在道术法主流程上，不替代其裁决；主流程始终拥有收口权。三类场景：

1. **重大决策 / 方案合议**：题面锁定、结论尚未最终冻结时，N 席围绕同一题面与待冻结结论各自独立判断，先互相质疑找反例 / 漏判 / 备路 / 不可接受结果，再交主流程收束。结构化提问与用户确认仍是必经门槛，合议只提高判断质量，不替代提问。
2. **待冻结方案的独立复查**：主路 / 边界 / 验收已形成待冻结结论时，至少 1 席（优先复核倾向模型 GPT / GLM）独立挑战，显式收口（采纳 / 部分采纳 / 驳回说明理由）后才可最终冻结。
3. **执行分工（按模型强弱与成本）**：冻结方案后，把可切分的执行子任务按上表路由到有当前证据支持的匹配模型——设计类可参考 Gemini，复核可参考 GPT / GLM；大体量低风险任务只取宿主已证实的低成本档，不凭模型名推断。每个执行席只承接已冻结题面、主路、边界与验收，单席位下发，不得擅改题面 / 主路 / 边界；主流程负责集成与一致性。

## 子 agent 下发包最小字段（每席一包）

- 冻结题面 / 子任务范围（不得改写）
- 本席位模型 + 选择理由
- 本席位关注点 / 切入点（须区别于主流程与其他席）
- 约束：沿用同版 `odai` 契约与 `references/dao/agent-governance.md`「子 agent 通用边界」（遇缺口停手回交、不自向用户提问、不派生子 agent、主流程复核才采纳）
- 输出格式与收口要求

## 分歧收口

- 各席结论冲突时，主流程不取多数票了事，按证据强弱与是否动摇道 / 术 / 边界裁决；足以动摇上游的，回写道与术重定。
- 收口后留一句：采纳了哪些席位的哪些点、驳回了什么、为什么。

## 能力不足时

- 激活了但宿主只给同系模型或给不全点名模型：先说明缺口，问用户按“能拿到的最高异构档”继续，还是退回 `references/dao/decision-challenge.md` 的自动 1 席兜底，不伪造多模型。
- 连并行 / 子 agent 都不暴露：退回主流程内显式反证一轮，并在结果总结写明未启用合议的原因。

## 回报

实际拉起的席位数、各席模型与分工、采纳 / 驳回结论，按 `references/dao/result-reporting.md` 如实回报；未真实派起的模型不得列入。
